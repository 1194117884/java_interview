## Build & Development Commands

- `npm run dev` ‚Äî Start Vite dev server at localhost:5173
- `npm run build` ‚Äî Full build: markdown data processing ‚Üí TypeScript check ‚Üí Vite production build
- `npm run build:data` ‚Äî Only regenerate JSON data from `../mds/` markdown files (no tsc/vite)
- `npm run preview` ‚Äî Preview production build locally

## Architecture

**Purpose**: Mobile-first Java interview question bank. Displays markdown-based Q&A content across 50+ topic categories. PWA ‚Äî installable and partially offline.

**Tech stack**: Vite 5 + React 18 + TypeScript + Tailwind CSS 3 + React Router v6. State via Zustand. Search via Fuse.js. Markdown rendering via react-markdown with Prism syntax highlighting.

### Data Pipeline

```
../mds/  (markdown files organized by category)
  ‚Üí scripts/build-data.js  (Node script, runs at build time)
  ‚Üí src/data/index.json, search-index.json  (imported at build for category/search index)
  ‚Üí public/data/index.json  (served as static JSON at runtime)
  ‚Üí public/data/content/{categoryId}.json  (per-category question JSON, fetched lazily)
```

The build script processes all `.md` files from `../mds/`, extracts titles from `# ` headings, sanitizes IDs, transforms relative image paths (`./img/...` ‚Üí `/mds/Category/...`), and writes JSON output.

### Routes

- `/` ‚Äî Home page with accordion-style `CategoryList`. Expands category on click, fetches `public/data/content/{categoryId}.json` at runtime (lazy loading).
- `/question/:categoryId/:questionId` ‚Äî Single question detail with prev/next navigation within the category.

### Key Components

| Component | Role |
|---|---|
| `CategoryList` | Accordion list of categories, lazy-loads questions on expand |
| `QuestionDetail` | Fetches category data, shows question content, prev/next buttons |
| `Header` | Fixed top bar with back button, search trigger, theme toggle |
| `SearchModal` | Modal overlay with Fuse.js-powered full-text search |
| `MarkdownRenderer` | Renders markdown as styled HTML with syntax-highlighted code blocks |
| `ThemeToggle` | Light/dark mode toggle |

### State (Zustand Store)

One store (`appStore`) with persistence for theme only: `theme`, `searchOpen`, `currentCategory`.

### Design System

Based on `DESIGN.md` in the project root ‚Äî an Anthropic Claude-inspired design system. Colors defined in `tailwind.config.js`:
- **Canvas**: #faf9f5 (light) / #181715 (dark)
- **Primary/coral**: #cc785c
- **Fonts**: Cormorant Garamond (display), Inter (body), JetBrains Mono (code)
- Dark mode via `data-theme="dark"` attribute on `<html>`

### Image Handling

Markdown images reference `../mds/{category}/img/...` relative paths. Build script rewrites them to `/mds/{category}/img/...`. Vite dev server has a custom middleware (`mdsPlugin`) that serves these files from `../mds/`. The build copies image directories to `dist/mds/`.

### PWA

Service worker generated by `vite-plugin-pwa` with `generateSW` strategy. Runtime caching for Google Fonts, images (CacheFirst) and navigation (NetworkFirst with offline fallback). Manifest in `public/manifest.json`.
