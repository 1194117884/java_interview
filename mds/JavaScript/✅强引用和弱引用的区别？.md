# ✅强引用和弱引用的区别？

# 典型回答

**强引用（Strong Reference）**和**弱引用（Weak Reference）** 的区别在于是否阻止垃圾回收器（GC）回收被引用的对象：

| 特性 | 强引用 | 弱引用 |
|------|:------:|:------:|
| 阻止GC回收 | 是—只要存在强引用，对象不会被回收 | 否—GC发现对象只有弱引用时立即回收 |
| 默认引用类型 | JavaScript中的默认引用方式 | 需要通过WeakMap/WeakSet/WeakRef创建 |
| 内存泄漏风险 | 高（容易产生泄漏） | 低（不阻止回收） |
| 常见用途 | 常规对象引用 | 缓存、元数据、DOM关联数据 |
| 可遍历性 | 可遍历 | 不可遍历（WeakMap/WeakSet） |

**核心理解：**
- **强引用**：**"这个对象必须留着"** — 只要还有强引用，对象就不会被GC回收
- **弱引用**：**"如果这个对象还在，我想用它；如果它被回收了也行"** — GC可以自由回收

# 扩展知识

## JavaScript中的弱引用实现

```javascript
// ES6引入的弱引用数据结构

// 1. WeakMap — 键是弱引用
const weakMap = new WeakMap();
let obj = { data: 'some data' };
weakMap.set(obj, 'metadata');
obj = null; // obj被GC回收后，weakMap中对应的条目自动删除

// 2. WeakSet — 值是弱引用
const weakSet = new WeakSet();
let obj2 = { data: 'more data' };
weakSet.add(obj2);
obj2 = null; // obj2被GC回收后，自动从weakSet中移除

// 3. WeakRef（ES2021）
let target = { name: 'target' };
const ref = new WeakRef(target);
console.log(ref.deref()?.name); // 'target' — 如果未被回收

target = null;
// 在某个时间点后
console.log(ref.deref()); // 可能是undefined（已被回收）或对象（尚未回收）
```

## WeakMap的应用场景

```javascript
// 1. 私有数据
const _secrets = new WeakMap();

class User {
  constructor(name, password) {
    this.name = name;
    _secrets.set(this, { password });
  }

  validate(inputPassword) {
    const { password } = _secrets.get(this) || {};
    return password === inputPassword;
  }
}

const user = new User('Alice', 'secret123');
// _secrets中的条目会随着user实例被回收而自动清理

// 2. DOM节点关联数据
const domData = new WeakMap();

function trackClick(element) {
  if (!domData.has(element)) {
    domData.set(element, { clicks: 0 });
  }
  element.addEventListener('click', () => {
    const data = domData.get(element);
    data.clicks++;
    console.log(`Clicked ${data.clicks} times`);
  });
}

// 当element被移除时，关联数据自动清理，无需手动清除

// 3. 缓存计算结果
const cache = new WeakMap();

function processExpensive(obj) {
  if (cache.has(obj)) {
    return cache.get(obj);
  }
  const result = heavyComputation(obj);
  cache.set(obj, result);
  return result;
}
// 不需要担心缓存增长，对象被回收时缓存自动清除

// 对比使用Map的内存泄漏风险
const badCache = new Map();
function processBad(obj) {
  if (badCache.has(obj)) return badCache.get(obj);
  const result = heavyComputation(obj);
  badCache.set(obj, result); // 即使obj被回收，Map中仍然持有引用
  return result;
}
```

## WeakSet的应用场景

```javascript
// 1. 标记/白名单
const processedItems = new WeakSet();

function processItem(item) {
  if (processedItems.has(item)) {
    return; // 已经处理过
  }
  // 处理item
  processedItems.add(item);
}

// 2. 防止实例被多次初始化
const initialized = new WeakSet();

class Connection {
  constructor(url) {
    if (initialized.has(this)) {
      throw new Error('Connection already initialized');
    }
    this.url = url;
    initialized.add(this);
  }
}

// 对比Set：Set会阻止对象被回收
```

## WeakRef 和 FinalizationRegistry

```javascript
// WeakRef（ES2021）— 更直接的弱引用
class ImageCache {
  constructor() {
    this.cache = new Map();
  }

  loadImage(url) {
    let ref = this.cache.get(url);

    if (ref) {
      const cached = ref.deref();
      if (cached) {
        console.log('从缓存加载');
        return cached;
      }
      console.log('缓存已被GC回收');
    }

    const img = new Image();
    img.src = url;
    // 存储弱引用
    this.cache.set(url, new WeakRef(img));
    return img;
  }

  // 清理已回收的引用
  cleanCache() {
    for (const [url, ref] of this.cache) {
      if (!ref.deref()) {
        this.cache.delete(url);
      }
    }
  }
}

// FinalizationRegistry — 监听对象被回收
const registry = new FinalizationRegistry((heldValue) => {
  console.log(`清理资源: ${heldValue}`);
});

class Resource {
  constructor(name) {
    this.name = name;
    this.handle = allocateResource(name);

    // 注册回调：当this被回收时调用
    registry.register(this, name);
  }

  release() {
    // 手动释放
    freeResource(this.handle);
    registry.unregister(this);
  }
}

function allocateResource(name) {
  console.log(`分配资源: ${name}`);
  return Symbol(name);
}

function freeResource(handle) {
  console.log(`释放资源: ${handle.toString()}`);
}

// 使用
let res = new Resource('file-handle');
res = null; // 当GC运行时，会打印 "清理资源: file-handle"
```

## 强引用与弱引用的对比

```javascript
// 强引用行为
let strongRef = { data: 'important' };
let anotherStrongRef = strongRef; // 两个强引用

strongRef = null; // 还有 anotherStrongRef 在引用，对象不会被回收

anotherStrongRef = null; // 此时没有强引用了，对象可被GC回收

// 弱引用行为
let obj3 = { data: 'temp' };
const weakRef = new WeakRef(obj3);

console.log(weakRef.deref()?.data); // 'temp'

obj3 = null;
// 下次GC运行后，weakRef.deref() 返回 undefined

// 关键区别：弱引用不会"延长"对象的生命周期
// 如果对象只剩下弱引用，GC会立即回收它
```

## 强引用导致的内存泄漏

```javascript
// 使用Map（强引用）导致的内存泄漏
const userSettings = new Map();

function getUserSettings(user) {
  if (!userSettings.has(user)) {
    userSettings.set(user, fetchSettings(user.id));
  }
  return userSettings.get(user);
}

let user = { id: 1, name: 'Alice' };
getUserSettings(user);

user = null; // user对象被置空
// 但userSettings中仍然存在user的引用
// user对象不能被GC回收！

// 使用WeakMap解决
const userSettingsWeak = new WeakMap();
// 当user对象被置空时，WeakMap中的条目自动清理
```

## 选择指南

| 场景 | 推荐 | 原因 |
|------|------|------|
| DOM节点关联数据 | WeakMap | DOM移除时自动清理 |
| 对象缓存 | WeakMap | 缓存不会阻止对象回收 |
| 对象标记集合 | WeakSet | 不影响对象的GC |
| 私有属性存储 | WeakMap | 与实例同生命周期 |
| 常规数据存储 | Map/Set | 需要遍历和size属性 |
| 公共数据缓存 | Map | 需要控制缓存生命周期 |
| 用户会话数据 | Map | 需要手动管理会话 |
| 事件监听器存储 | WeakMap | 与DOM元素生命周期绑定 |

## 注意事项

```javascript
// 1. WeakRef的deref()结果不可预测
function useWeakRef() {
  const ref = new WeakRef({});
  const obj = ref.deref();
  // obj可能在任何时候变成undefined
  // 不能假设obj会存在
}

// 2. FinalizationRegistry的回调时机不确定
// 回调可能在主线程或GC线程执行
// 不要在回调中执行关键操作

// 3. WeakMap不可迭代
const wm = new WeakMap();
// wm.keys() // TypeError
// wm.values() // TypeError
// wm.size // undefined

// 4. 基本类型不能作为弱引用键
// wm.set(42, 'value') // TypeError: Invalid value used as weak map key

// 5. 弱引用不应该被滥用
// 大多数场景使用强引用即可
// 弱引用适用于：缓存、元数据、DOM关联等特殊场景
```
