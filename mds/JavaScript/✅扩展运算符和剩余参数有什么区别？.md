# ✅扩展运算符和剩余参数有什么区别？

# 典型回答

扩展运算符（Spread Operator）和剩余参数（Rest Parameters）都使用三个点`...`作为前缀，但它们的作用和使用场景正好相反：

**核心区别：**

| 对比维度 | 扩展运算符 (...) | 剩余参数 (...) |
|---------|----------------|---------------|
| 使用位置 | 函数调用或字面量中 | 函数参数定义中 |
| 作用 | 将可迭代对象"展开"为独立元素 | 将多个参数"收集"为数组 |
| 展开/收集 | 展开（拆散） | 收集（聚合） |
| 出现位置 | 在数组/对象字面量、函数调用中作为值 | 在函数参数列表的最后一个位置 |

**简单记忆：**
- **剩余参数**：在**函数定义**时使用，将不确定的参数收集成一个数组
- **扩展运算符**：在**函数调用**或**字面量**中使用，将数组或对象展开成单个元素

# 扩展知识

## 剩余参数详解

```javascript
// 基本用法：收集剩余参数
function sum(...numbers) {
  return numbers.reduce((total, n) => total + n, 0);
}
console.log(sum(1, 2, 3, 4)); // 10

// 与普通参数结合（剩余参数必须在最后）
function log(name, role, ...scores) {
  console.log(`${name} (${role}): ${scores.join(', ')}`);
}
log('Alice', 'Student', 95, 88, 92);
// Alice (Student): 95, 88, 92

// 代替arguments对象
function oldWay() {
  console.log(arguments.length); // 类数组，不是真正的数组
}
function newWay(...args) {
  console.log(args.length); // 真正的数组
}

// 箭头函数中的剩余参数（唯一获得可变参数的方式）
const arrowSum = (...nums) => nums.reduce((a, b) => a + b, 0);
```

## 扩展运算符详解

```javascript
// 1. 展开数组
const arr1 = [1, 2, 3];
const arr2 = [4, 5, 6];
const combined = [...arr1, ...arr2]; // [1, 2, 3, 4, 5, 6]

// 2. 复制数组（浅拷贝）
const original = [1, 2, [3, 4]];
const clone = [...original];
clone[2][0] = 99;
console.log(original[2][0]); // 99 — 浅拷贝，嵌套对象仍共享引用

// 3. 展开对象（ES2018）
const obj1 = { a: 1, b: 2 };
const obj2 = { c: 3, ...obj1 }; // { c: 3, a: 1, b: 2 }
console.log(obj2);

// 4. 合并对象（后面的属性会覆盖前面的）
const merged = { ...obj1, ...{ b: 99, d: 4 } };
console.log(merged); // { a: 1, b: 99, d: 4 }

// 5. 展开字符串
const chars = [...'hello'];
console.log(chars); // ['h', 'e', 'l', 'l', 'o']

// 6. 展开Set/Map
const set = new Set([1, 2, 3]);
const arr = [...set]; // [1, 2, 3]

// 7. 函数调用中展开
const nums = [3, 1, 4, 1, 5];
console.log(Math.max(...nums)); // 5
```

## 深入对比

```javascript
// 剩余参数 — 在函数定义时收集
function show(first, ...rest) {
  // rest 是数组 [2, 3, 4]
}
show(1, 2, 3, 4);

// 扩展运算符 — 在函数调用时展开
const args = [2, 3, 4];
show(1, ...args); // 等价于 show(1, 2, 3, 4)

// 两者配合使用
function multiply(multiplier, ...numbers) {
  return numbers.map(n => n * multiplier);
}
const factors = [2, 3, 4];
console.log(multiply(10, ...factors)); // [20, 30, 40]
```

## 剩余参数与arguments对象的区别

```javascript
function compare() {
  // arguments
  console.log(Array.isArray(arguments)); // false
  console.log(Object.prototype.toString.call(arguments)); // [object Arguments]

  // 剩余参数
  const rest = [...arguments];
  console.log(Array.isArray(rest)); // true
}

// arguments的限制：
// 1. 类数组，不是真正的数组
// 2. 箭头函数中没有arguments
// 3. 不能在严格模式与默认参数混用（旧规范）
function example(a, b = 2) {
  'use strict';
  // console.log(arguments); // 严格模式下与默认参数一起使用会报错
}
```

## 扩展运算符实现原理

数组的扩展运算符基于Iterator接口：

```javascript
// 等价实现
function spreadArray(iterable) {
  const items = [];
  const iterator = iterable[Symbol.iterator]();
  let next = iterator.next();
  while (!next.done) {
    items.push(next.value);
    next = iterator.next();
  }
  return items;
}

// 没有Iterator接口的对象不能展开
const obj = { a: 1, b: 2 };
// console.log(...obj); // TypeError: Found non-callable @@iterator

// 对象扩展运算符（ES2018）使用不同的机制——基于可枚举属性
```

## 常见陷阱

```javascript
// 1. 剩余参数必须是最后一个
// function bad(a, ...rest, b) {} // SyntaxError

// 2. 构造函数的剩余参数
class CustomArray extends Array {
  constructor(...args) {
    super(...args);
  }
}

// 3. 扩展运算符进行浅拷贝
const original = { nested: { value: 1 } };
const copy = { ...original };
copy.nested.value = 2;
console.log(original.nested.value); // 2 — 浅拷贝！

// 4. 对象扩展运算符只展开可枚举属性
const obj = { a: 1 };
Object.defineProperty(obj, 'b', { value: 2, enumerable: false });
console.log({ ...obj }); // { a: 1 }
```

## 解构中的剩余模式

```javascript
// 数组解构中的剩余模式
const [first, ...rest] = [1, 2, 3, 4];
console.log(first, rest); // 1 [2, 3, 4]

// 对象解构中的剩余模式
const { a, ...others } = { a: 1, b: 2, c: 3 };
console.log(a, others); // 1 { b: 2, c: 3 }

// 注意：解构中的剩余必须是最后一个
// const [...rest, last] = [1,2,3]; // SyntaxError

// 在函数参数解构中使用
function process({ name, ...details }) {
  console.log(name, details);
}
process({ name: 'Alice', age: 25, role: 'dev' });
// Alice { age: 25, role: 'dev' }
```
