# ✅严格模式有哪些限制？

# 典型回答

**严格模式（Strict Mode）** 是ES5引入的一种更严格的JavaScript执行模式，通过消除语言中一些不合理、不严谨的方面，提高代码安全性和性能。

**开启方式：**
```javascript
// 全局严格模式
'use strict';

// 函数级严格模式
function strictFunction() {
  'use strict';
  // 函数内部严格模式
}

// ES Module 和 Class 默认开启严格模式
// 不需要显式声明
```

**严格模式的主要限制：**

| 类别 | 限制 | 说明 |
|------|------|------|
| 变量 | 不允许未声明的变量赋值 | `x = 1` 抛出ReferenceError |
| 变量 | 不允许删除变量 | `delete x` 抛出SyntaxError |
| 变量 | 不允许重复参数名 | `function(a, a) {}` 报错 |
| 对象 | 不允许删除不可配置属性 | `delete Object.prototype` 报错 |
| 对象 | 不允许给只读属性赋值 | `NaN = 1` 报错 |
| this | this为undefined而不是全局对象 | 函数独立调用时 |
| 函数 | 不允许arguments.callee | 抛出TypeError |
| 函数 | arguments不追踪参数变化 | 参数与arguments无关联 |
| 函数 | eval有独立作用域 | eval内声明的变量不污染外部 |
| 语言 | 不允许with语句 | 抛出SyntaxError |
| 语言 | 不允许八进制字面量 | `0123` 报错 |
| 安全 | 不允许使用implements等保留字 | 保留字不能用作标识符 |

# 扩展知识

## 变量相关限制

```javascript
'use strict';

// 1. 未声明变量赋值报错
// x = 10; // ReferenceError: x is not defined
let x = 10; // 正确

// 2. 删除变量报错
let y = 20;
// delete y; // SyntaxError: Delete of an unqualified identifier

// 3. 重复参数名报错
// function sum(a, a, c) { // SyntaxError: Duplicate parameter name not allowed
//   return a + a + c;
// }

// 4. 使用未来的保留字
// let implements; // SyntaxError: Unexpected strict mode reserved word
// let interface;
// let package;
// let private;
// let protected;
// let public;
// let static;
// let yield;
```

## this 的严格模式行为

```javascript
'use strict';

// 1. 独立函数调用中的this为undefined
function showThis() {
  console.log(this); // undefined
}
showThis();

// 2. 全局作用域的this
console.log(this); // window（模块中为undefined）

// 3. setTimeout中的this
setTimeout(function() {
  console.log(this); // undefined（而非window）
}, 100);

// 4. 事件处理中的this
button.addEventListener('click', function() {
  console.log(this); // 仍然是触发事件的元素
});

// 5. call/apply中的基本类型不会被包装
function showType() {
  console.log(typeof this);
}
showType.call(42);      // 'number'（非严格模式是'object', Number对象）
showType.call('hello'); // 'string'
showType.call(true);    // 'boolean'

// 6. null/undefined作为thisArg
function strictNull() {
  'use strict';
  console.log(this); // null
}
strictNull.call(null); // null（非严格模式是window）
```

## arguments 对象的限制

```javascript
'use strict';

// 1. arguments与参数不再互相影响
function strictArgs(a, b) {
  a = 100;
  console.log(arguments[0]); // 1 — 不随a变化
  console.log(a);            // 100
}
strictArgs(1, 2);

// 非严格模式对比
function nonStrictArgs(a, b) {
  a = 100;
  console.log(arguments[0]); // 100 — 随a变化
}
nonStrictArgs(1, 2);

// 2. arguments.callee和arguments.caller报错
function strictCallee() {
  // arguments.callee(); // TypeError: 'caller', 'callee' access is restricted
}

// 3. 不能删除arguments
// delete arguments; // SyntaxError
```

## eval 的严格模式限制

```javascript
'use strict';

// eval在严格模式中创建独立作用域
function strictEval() {
  eval('var localVar = 10; let localLet = 20;');
  // console.log(localVar); // ReferenceError（不污染外部）
  // console.log(localLet); // ReferenceError（不污染外部）
}

// 非严格模式对比
function nonStrictEval() {
  eval('var localVar = 10;');
  console.log(localVar); // 10 — 污染了外部作用域
}

// eval本身在严格模式代码中也被视为严格模式
function callEval() {
  'use strict';
  eval('var x = 1; with({}) {}'); // with在严格模式eval中也报错
}
```

## 删除操作的限制

```javascript
'use strict';

// 1. 不能删除普通变量
let a = 1;
// delete a; // SyntaxError

// 2. 不能删除函数
function fn() {}
// delete fn; // SyntaxError

// 3. 不能删除不可配置的属性
// delete Object.prototype; // TypeError
// delete Array.prototype; // TypeError

// 4. 可以删除对象属性（如果configurable为true）
const obj = {};
Object.defineProperty(obj, 'prop', {
  value: 1,
  configurable: false
});
// delete obj.prop; // TypeError（不可配置）
```

## 对象字面量的限制

```javascript
'use strict';

// 1. 不能有重复的属性名（ES5规范）
// const obj = { x: 1, x: 2 }; // ES5中SyntaxError
// ES6中允许覆盖，不报错

// 2. 给只读属性赋值报错
const obj = {};
Object.defineProperty(obj, 'readOnly', {
  value: 42,
  writable: false
});
// obj.readOnly = 100; // TypeError: Cannot assign to read only property

// 3. 给不可扩展对象添加属性报错
const fixed = Object.preventExtensions({});
// fixed.newProp = 1; // TypeError: Cannot add property
```

## 八进制字面量的限制

```javascript
'use strict';

// 八进制字面量
// var x = 0123; // SyntaxError: Octal literals are not allowed

// ES6中可以用0o前缀
const x = 0o123; // 83（十进制）
console.log(x); // 83

// 八进制转义字符在字符串中也不允许
// var str = '\123'; // SyntaxError: Octal escape sequences are not allowed
```

## 其他限制

```javascript
'use strict';

// 1. with语句禁止
// with (Math) {
//   console.log(PI); // SyntaxError
// }

// 2. 禁止修改原始值的属性
// (1).prop = 'test'; // TypeError: Cannot create property
// 'hello'.prop = 'test'; // TypeError

// 3. 更严格的变量检查
// eval 和 arguments 不能用作标识符
// let eval = 1; // SyntaxError
// let arguments = 2; // SyntaxError

// 4. 函数声明必须在顶层或函数体
if (true) {
  // function inner() {} // ES6中允许但行为不同
}

// 5. 变量不能未声明
// 好处：提前发现bug
```

## 严格模式的好处

| 好处 | 说明 |
|------|------|
| 错误及时暴露 | 将静默失败转为显式异常 |
| 提高安全性 | 禁止eval修改作用域、禁止with |
| 性能优化 | 引擎可以做出更多优化假设 |
| 便于调试 | 避免意外全局变量等难以追踪的问题 |
| 为ES未来版本准备 | 限制了一些不规范的语法特性 |

## 默认使用严格模式的场景

```javascript
// 1. ES Module — 默认严格模式
// module.js
export function test() {
  // 这里已经是严格模式
  console.log(this); // undefined
}

// 2. Class — 默认严格模式
class MyClass {
  constructor() {
    // 严格模式
  }
  method() {
    // 严格模式
    console.log(this); // undefined（如果独立调用）
  }
}

// 3. 使用class语法定义的类的方法
```

## 最佳实践

- 新代码始终使用严格模式
- ES Module和class中不需要显式声明（默认严格模式）
- 在脚本文件开头添加 `'use strict'`
- 使用打包工具时通常会自动添加严格模式
- TypeScript编译后默认使用严格模式
