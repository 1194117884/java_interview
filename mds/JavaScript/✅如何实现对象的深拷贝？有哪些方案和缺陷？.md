# ✅如何实现对象的深拷贝？有哪些方案和缺陷？

# 典型回答

深拷贝（Deep Copy）是指创建一个新对象，递归复制原对象所有层级的数据，新对象与原对象**完全独立**，互不影响。与之相对，浅拷贝（Shallow Copy）只复制第一层属性，嵌套对象仍共享引用。

**常见深拷贝方案：**

| 方案 | 适用场景 | 缺陷 |
|------|---------|------|
| JSON.parse(JSON.stringify(obj)) | 简单对象，无特殊类型 | 无法处理函数、undefined、Symbol、循环引用、Date、RegExp等 |
| 递归手写深拷贝 | 需要自定义控制 | 需要自行处理各种边界情况 |
| structuredClone() | 现代浏览器/Node 17+ | 不支持函数和Symbol |
| lodash.cloneDeep() | 生产环境 | 需要引入第三方库 |
| MessageChannel | 部分特殊场景 | 异步、不支持函数 |

# 扩展知识

## JSON.parse/stringify 方案的缺陷

```javascript
const obj = {
  name: 'Alice',
  age: undefined,        // 丢失
  sayHi: function() {},  // 丢失
  sym: Symbol('key'),    // 丢失
  date: new Date(),      // 变成字符串
  reg: /test/i,          // 变成空对象{}
  map: new Map(),        // 变成空对象{}
  set: new Set(),        // 变成空对象{}
  nan: NaN,              // 变成 null
  inf: Infinity,         // 变成 null
  negInf: -Infinity,     // 变成 null
};

const cloned = JSON.parse(JSON.stringify(obj));
console.log(cloned);
// { name: 'Alice', date: '2024-...', reg: {}, map: {}, set: {}, nan: null, ... }

// 循环引用会抛出错误
const circular = { self: null };
circular.self = circular;
// JSON.stringify(circular); // TypeError: Converting circular structure to JSON
```

## 手动实现深拷贝

```javascript
function deepClone(value, map = new WeakMap()) {
  // 处理基本类型
  if (value === null || typeof value !== 'object') return value;

  // 处理循环引用
  if (map.has(value)) return map.get(value);

  // 处理特殊对象类型
  if (value instanceof Date) return new Date(value);
  if (value instanceof RegExp) return new RegExp(value.source, value.flags);
  if (value instanceof Map) {
    const cloned = new Map();
    map.set(value, cloned);
    value.forEach((v, k) => cloned.set(deepClone(k, map), deepClone(v, map)));
    return cloned;
  }
  if (value instanceof Set) {
    const cloned = new Set();
    map.set(value, cloned);
    value.forEach(v => cloned.add(deepClone(v, map)));
    return cloned;
  }
  if (value instanceof WeakMap || value instanceof WeakSet) return value;
  if (ArrayBuffer.isView(value)) return value.slice(0);
  if (value instanceof Promise) return value; // Promise不可转移

  // 处理普通对象和数组
  const cloned = Array.isArray(value) ? [] : {};
  map.set(value, cloned);

  // 复制所有属性（包括Symbol属性）
  for (const key of [...Object.keys(value), ...Object.getOwnPropertySymbols(value)]) {
    cloned[key] = deepClone(value[key], map);
  }

  return cloned;
}

// 测试
const obj = {
  num: 1,
  str: 'hello',
  bool: true,
  null: null,
  undefined: undefined,
  arr: [1, { a: 2 }],
  obj: { x: 1, y: { z: 2 } },
  date: new Date('2024-01-01'),
  regex: /test/gi,
  map: new Map([['key', 'value']]),
  set: new Set([1, 2, 3]),
  [Symbol('sym')]: 'symbol value',
  nested: { deep: { deeper: { value: 42 } } }
};

const cloned = deepClone(obj);
console.log(cloned.date instanceof Date); // true
console.log(cloned.regex instanceof RegExp); // true
console.log(cloned.arr !== obj.arr); // true
console.log(cloned.nested.deep.deeper !== obj.nested.deep.deeper); // true
```

## structuredClone() 方案

```javascript
// Node 17+ / 现代浏览器内置
const obj = {
  name: 'Alice',
  scores: [95, 88, 92],
  address: { city: 'Beijing' },
  date: new Date(),
  reg: /test/i,
  map: new Map([['key', 'value']]),
  set: new Set([1, 2, 3]),
  blob: new Blob(['hello']),
};

const cloned = structuredClone(obj);
console.log(cloned.date instanceof Date);      // true
console.log(cloned.reg instanceof RegExp);      // true
console.log(cloned.map instanceof Map);         // true
console.log(cloned.set instanceof Set);         // true

// 仍然不支持
// const obj2 = { fn: () => {}, sym: Symbol() };
// structuredClone(obj2); // DataCloneError
```

## lodash.cloneDeep

```javascript
// 生产环境最完善的方案
import _ from 'lodash';

const obj = { /* 任意复杂对象 */ };
const cloned = _.cloneDeep(obj);

// lodash的实现会处理：
// - 所有基本类型
// - 数组和对象
// - Date、RegExp、Map、Set、Buffer
// - 循环引用
// - 原型链
// - 各种边界情况
```

## MessageChannel 方案

```javascript
// 利用结构化克隆算法（异步）
function structuralClone(obj) {
  return new Promise(resolve => {
    const { port1, port2 } = new MessageChannel();
    port2.onmessage = ev => resolve(ev.data);
    port1.postMessage(obj);
  });
}

// 使用
const obj = { /* ... */ };
structuralClone(obj).then(cloned => {
  console.log(cloned);
});
```

## 浅拷贝回顾

```javascript
// 浅拷贝方式
const obj = { a: 1, nested: { b: 2 } };

// 1. 扩展运算符
const spread = { ...obj };

// 2. Object.assign
const assigned = Object.assign({}, obj);

// 3. 数组的slice/concat/from
const arr = [1, { a: 2 }];
const sliced = arr.slice();
const concated = arr.concat();
const fromArr = Array.from(arr);

// 浅拷贝的陷阱
spread.nested.b = 99;
console.log(obj.nested.b); // 99 — 互相影响！
```

## 深拷贝的性能考量

```javascript
// 对于大型对象，深拷贝很耗时
const largeObj = {};
for (let i = 0; i < 100000; i++) {
  largeObj[`key_${i}`] = { value: i, nested: { data: `string_${i}` } };
}

console.time('JSON');
const jsonClone = JSON.parse(JSON.stringify(largeObj));
console.timeEnd('JSON');

console.time('structuredClone');
const scClone = structuredClone(largeObj);
console.timeEnd('structuredClone');

// structuredClone 通常比 JSON 快 2-5 倍
```

## 选择指南

- **日常简单对象**：`JSON.parse(JSON.stringify(obj))`（确认无特殊类型）
- **现代环境（Node 17+/Chrome 98+）**：`structuredClone()`
- **生产级完整方案**：`lodash.cloneDeep()`
- **需要深度控制**：手写深拷贝函数
- **性能优先**：`structuredClone()` 或 MessageChannel
