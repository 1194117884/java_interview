# ✅宏任务和微任务有哪些？执行顺序是怎样的？

# 典型回答

## 宏任务（MacroTask）和微任务（MicroTask）列表

**宏任务（MacroTask）：**

| 来源 | 具体API |
|------|---------|
| 主代码块 | script整体代码 |
| 定时器 | setTimeout、setInterval |
| I/O | DOM事件、网络请求回调、文件操作 |
| 浏览器专用 | requestAnimationFrame（介于宏任务和渲染之间） |
| Node.js 专用 | setImmediate、I/O回调 |

**微任务（MicroTask）：**

| 来源 | 具体API |
|------|---------|
| Promise | Promise.then/catch/finally |
| MutationObserver | DOM变化观察 |
| queueMicrotask | 显式创建微任务 |
| Node.js 专用 | process.nextTick |

## 执行顺序规则

```
1. 执行一个宏任务（从script开始）
2. 执行所有微任务（清空微任务队列）
3. 执行下一个宏任务
4. 重复步骤2-3

在一次循环中：一个宏任务 → 所有微任务 → 可能渲染 → 下一个宏任务
```

**核心原则：**
- 每个宏任务执行完后，都会清空微任务队列
- 微任务中产生的新微任务会继续在当前循环中执行
- 微任务队列总是在下一个宏任务之前被清空

# 扩展知识

## 基本示例

```javascript
console.log('1: script start');

setTimeout(() => {
  console.log('2: setTimeout');
}, 0);

Promise.resolve().then(() => {
  console.log('3: promise1');
});

Promise.resolve().then(() => {
  console.log('4: promise2');
});

console.log('5: script end');

// 输出顺序：
// 1: script start
// 5: script end
// 3: promise1
// 4: promise2
// 2: setTimeout
```

## 微任务递归的情况

```javascript
// 微任务中创建微任务——会继续在当前循环中执行
Promise.resolve().then(() => {
  console.log('micro 1');
  Promise.resolve().then(() => {
    console.log('micro 2');
    Promise.resolve().then(() => {
      console.log('micro 3');
    });
  });
});

setTimeout(() => {
  console.log('task 1');
}, 0);

// 输出：
// micro 1
// micro 2
// micro 3
// task 1

// 微任务队列会在宏任务之前被完全清空
```

## 宏任务与微任务的嵌套

```javascript
setTimeout(() => {
  console.log('task1');
  Promise.resolve().then(() => {
    console.log('micro in task1');
  });
}, 0);

setTimeout(() => {
  console.log('task2');
  Promise.resolve().then(() => {
    console.log('micro in task2');
  });
}, 0);

// 输出：
// task1
// micro in task1
// task2
// micro in task2

// 每个宏任务后都会清空微任务队列
```

## 复杂综合示例

```javascript
console.log('1');

setTimeout(() => {
  console.log('2');
  Promise.resolve().then(() => console.log('3'));
}, 0);

new Promise((resolve) => {
  console.log('4');
  resolve();
}).then(() => {
  console.log('5');
  setTimeout(() => console.log('6'), 0);
  Promise.resolve().then(() => console.log('7'));
});

setTimeout(() => {
  console.log('8');
  Promise.resolve().then(() => console.log('9'));
}, 0);

console.log('10');

// 逐步分析：
// 宏任务1（script）:
//   console.log('1') → 1
//   setTimeout(2) → 宏任务队列: [2]
//   Promise执行器: console.log('4') → 4, resolve()
//   .then(5) → 微任务队列: [5]
//   setTimeout(8) → 宏任务队列: [2, 8]
//   console.log('10') → 10
//   输出: 1, 4, 10
//
// 清空微任务:
//   .then(5): console.log('5') → 5
//            setTimeout(6) → 宏任务队列: [2, 8, 6]
//            Promise.resolve().then(7) → 微任务队列: [7]
//   执行微任务7: console.log('7') → 7
//   输出: 5, 7
//
// 宏任务2（setTimeout2）:
//   console.log('2') → 2
//   Promise.resolve().then(3) → 微任务队列: [3]
//   清空微任务: console.log('3') → 3
//   输出: 2, 3
//
// 宏任务3（setTimeout8）:
//   console.log('8') → 8
//   Promise.resolve().then(9) → 微任务队列: [9]
//   清空微任务: console.log('9') → 9
//   输出: 8, 9
//
// 宏任务4（setTimeout6）:
//   console.log('6') → 6
//   输出: 6

// 最终输出: 1, 4, 10, 5, 7, 2, 3, 8, 9, 6
```

## Node.js 中的差异

```javascript
// Node.js 中 process.nextTick 优先级高于 Promise 微任务
process.nextTick(() => {
  console.log('nextTick');
});

Promise.resolve().then(() => {
  console.log('promise');
});

// Node.js 输出：nextTick → promise

// 执行顺序：nextTick队列 → 微任务队列 → 宏任务

// process.nextTick 实际上是"nextTickQueue"，比微任务更优先
// 可以创建"深度递归"
function recursiveNextTick() {
  process.nextTick(() => {
    console.log('nextTick');
    recursiveNextTick(); // 无限递归
  });
}
// 这会导致I/O任务被"饿死"
```

## queueMicrotask 的使用

```javascript
// 显式创建微任务（代替Promise.resolve().then()）
queueMicrotask(() => {
  console.log('microtask');
});

// 相当于
Promise.resolve().then(() => {
  console.log('microtask');
});

// 使用场景：需要在当前同步代码执行完毕后、渲染前执行回调
function measurePerformance() {
  queueMicrotask(() => {
    console.log('同步代码执行完毕');
    // 测量性能
  });
}
```

## requestAnimationFrame 的位置

```javascript
// requestAnimationFrame 的特殊位置
// 执行顺序：宏任务 → 微任务 → requestAnimationFrame → 渲染 → requestIdleCallback

console.log('1: start');

setTimeout(() => {
  console.log('2: setTimeout');
}, 0);

requestAnimationFrame(() => {
  console.log('3: rAF');
});

Promise.resolve().then(() => {
  console.log('4: promise');
});

console.log('5: end');

// 通常输出：
// 1, 5, 4, 3, 2
// 或者
// 1, 5, 4, 2, 3

// 注意：setTimeout和rAF的顺序在不同浏览器可能不同
// 但微任务（promise）总是在两者之前
```

## 浏览器与Node.js的事件循环对比

| 阶段 | 浏览器 | Node.js |
|------|--------|---------|
| 微任务优先级 | 每次宏任务后清空 | 每个阶段后清空 |
| process.nextTick | 不存在 | 比Promise更高优先级 |
| setImmediate | 不存在 | 在check阶段执行 |
| 渲染时机 | 微任务后、下一个宏任务前 | 无渲染概念 |
| 任务队列 | 宏任务和微任务两个队列 | 多个阶段队列 |

## 实际应用

```javascript
// 1. 在微任务中批量更新
let pendingUpdates = [];
let scheduled = false;

function scheduleUpdate(update) {
  pendingUpdates.push(update);
  if (!scheduled) {
    scheduled = true;
    queueMicrotask(() => {
      const updates = pendingUpdates;
      pendingUpdates = [];
      scheduled = false;
      // 批量处理所有更新
      updates.forEach(fn => fn());
    });
  }
}

// 2. 分割长任务
function processLargeData(items) {
  let index = 0;

  function processChunk() {
    const start = performance.now();
    while (index < items.length && performance.now() - start < 16) {
      processItem(items[index++]);
    }
    if (index < items.length) {
      // 使用宏任务让出主线程
      setTimeout(processChunk, 0);
    }
  }

  setTimeout(processChunk, 0);
}
```

## 最佳实践

- **避免在微任务中创建无限循环**（会阻塞主线程）
- **UI更新相关的操作放在宏任务中**（给渲染留机会）
- **高优先级任务用 queueMicrotask**
- **长任务分割用 setTimeout**
- 理解执行顺序有助于调试异步bug
