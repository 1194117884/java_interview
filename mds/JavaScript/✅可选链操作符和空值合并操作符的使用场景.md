# ✅可选链操作符和空值合并操作符的使用场景

# 典型回答

**可选链操作符（Optional Chaining）`?.`** 和**空值合并操作符（Nullish Coalescing）`??`** 是ES2020引入的两个语法特性，用于更安全、更简洁地处理null/undefined值。

**可选链 `?.`**：在访问深层嵌套的属性时，如果中间的某个引用是null或undefined，表达式会短路返回undefined而不是抛出错误。

```javascript
// 传统写法
const city = user && user.address && user.address.city;

// 可选链写法
const city = user?.address?.city;
```

**空值合并 `??`**：当左侧表达式的结果是null或undefined时，返回右侧的默认值。
```javascript
const name = input ?? 'default';
```

**对比 `||` 和 `??`：**
- `||`：遇到所有**假值**（false、0、''、null、undefined、NaN）都返回右侧
- `??`：仅当值为**null或undefined**时才返回右侧

# 扩展知识

## 可选链的三种形式

```javascript
// 1. 静态属性访问
const name = obj?.name;

// 2. 动态属性访问（方括号）
const key = 'name';
const value = obj?.[key];

// 3. 方法调用
const result = obj?.method?.();
```

## 可选链的短路行为

```javascript
const obj = null;

// 短路：a?.b 已经返回 undefined，后续不会执行
const result = obj?.a?.b?.c;  // undefined，不会报错

// 方法调用的短路
const arr = null;
arr?.[0]?.();  // undefined，不会尝试调用

// 短路也影响赋值
let x = 1;
const y = null?.prop = x++;  // 短路，x++不会执行
console.log(x);  // 1
```

## 常见使用场景

```javascript
// 1. API响应解析
const response = await fetch('/api/user');
const userName = response?.data?.user?.name ?? 'Anonymous';

// 2. 深层配置对象
const config = {};
const timeout = config?.server?.connection?.timeout ?? 3000;

// 3. 可选的方法调用
// 事件回调可能不存在
onSuccess?.();

// DOM元素可能不存在
const element = document.querySelector('.not-exists');
element?.textContent?.trim();  // 安全

// 4. 数组访问
const arr = [1, 2, 3];
console.log(arr?.[10]);  // undefined（超出范围）

// 5. 动态导入
const module = await import('./module')?.default;

// 6. 配合delete
delete obj?.nested?.prop;
```

## 空值合并操作符详解

```javascript
// null和undefined触发默认值
console.log(null ?? 'default');     // 'default'
console.log(undefined ?? 'default'); // 'default'

// 其他假值不会触发默认值
console.log(0 ?? 'default');       // 0
console.log('' ?? 'default');      // ''
console.log(false ?? 'default');   // false
console.log(NaN ?? 'default');     // NaN

// 对比 ||
console.log(0 || 'default');       // 'default'
console.log('' || 'default');      // 'default'
console.log(false || 'default');   // 'default'
```

## 空值合并的常见场景

```javascript
// 1. 数值0的合法值
function setVolume(volume) {
  this.volume = volume ?? 50;  // 允许0作为合法值
}
setVolume(0);   // 0（??）
setVolume(0);   // 50（||）错误！

// 2. 空字符串
function setLabel(label) {
  this.label = label ?? 'Untitled';  // 允许''作为合法值
}

// 3. 布尔值false
const showFeature = flag ?? true;  // 允许false作为合法值
```

## 结合使用的模式

```javascript
// 安全获取深层属性+提供默认值
const userName = user?.profile?.name ?? 'Guest';

// 链式默认值
const timeout = config?.timeout ?? retryConfig?.timeout ?? 3000;

// 函数调用默认值
const result = expensiveCalc?.() ?? fallback();

// 数组操作
const first = arr?.[0] ?? 'empty';
```

## 注意事项和陷阱

```javascript
// 1. 不能用于赋值左侧
// obj?.prop = 1; // SyntaxError

// 2. 可选链用在未声明变量时仍会报错
// notDeclared?.prop // ReferenceError: notDeclared is not defined

// 3. 短路可能导致副作用意外跳过
let count = 0;
const result = obj?.method?.(count++);  // 如果obj?.method是nullish，count++不会执行

// 4. 可选链不能与括号混用
// (obj?.prop)?.method // 可以，但通常不需要

// 5. 与空值合并的优先级
// 需要括号明确优先级
const val = a ?? (b ?? c);     // 明确先计算b ?? c
const val2 = (a?.b) ?? 'default'; // 推荐写法

// 6. 空值合并不能与&&或||直接混用
// const x = a ?? b || c;  // SyntaxError
const x = (a ?? b) || c;  // 需要括号
```

## delete操作符与可选链

```javascript
const obj = { nested: { prop: 1 } };

// 安全删除嵌套属性
delete obj?.nested?.prop;
console.log(obj);  // { nested: {} }

// 如果中间是null，delete不会执行
const obj2 = { nested: null };
delete obj2?.nested?.prop;  // 无操作，不报错
```

## 在TypeScript中的意义

```javascript
// TypeScript中，可选链配合严格空检查非常有用
interface User {
  name?: string;
  address?: {
    city?: string;
  }
}

function getCity(user: User): string {
  return user?.address?.city ?? 'Unknown';
}
```

## 浏览器兼容性

可选链和空值合并操作符在现代浏览器（Chrome 80+、Firefox 74+、Safari 13.1+）中得到支持。在较老的环境中需要通过Babel等工具转译。转译后代码会变成三元表达式或逻辑与运算的形式。
