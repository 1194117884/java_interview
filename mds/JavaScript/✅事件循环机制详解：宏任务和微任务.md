# ✅事件循环机制详解：宏任务和微任务

# 典型回答

**事件循环（Event Loop）** 是JavaScript的异步执行机制，允许JavaScript在单线程环境下处理异步操作。

JavaScript是**单线程**的，但通过事件循环实现了**非阻塞I/O**。事件循环的核心是**宏任务（MacroTask）** 和**微任务（MicroTask）** 的队列管理。

**执行顺序：**

1. 执行当前宏任务（从Script代码开始）
2. 执行所有微任务（清空微任务队列）
3. 执行下一个宏任务
4. 重复步骤2-3

```
宏任务 → 微任务队列 → 宏任务 → 微任务队列 → ...
```

**常见的任务类型：**

| 任务类型 | 举例 |
|---------|------|
| 宏任务（MacroTask） | script整体代码、setTimeout、setInterval、setImmediate(Node)、I/O、UI渲染 |
| 微任务（MicroTask） | Promise.then/catch/finally、MutationObserver、process.nextTick(Node) |

# 扩展知识

## 基本执行流程

```javascript
console.log('1: script start');

setTimeout(function() {
  console.log('2: setTimeout');
}, 0);

Promise.resolve()
  .then(function() {
    console.log('3: promise1');
  })
  .then(function() {
    console.log('4: promise2');
  });

console.log('5: script end');

// 执行顺序：
// 1: script start
// 5: script end
// 3: promise1
// 4: promise2
// 2: setTimeout

// 流程分析：
// 1. 执行宏任务（script整体代码）
//    - 输出 '1: script start'
//    - setTimeout注册，放入宏任务队列
//    - Promise.then注册，放入微任务队列
//    - 输出 '5: script end'
// 2. 清空微任务队列
//    - 执行第一个then，输出 '3: promise1'
//    - 返回新Promise，第二个then注册到微任务队列
//    - 执行第二个then，输出 '4: promise2'
// 3. 取下一个宏任务
//    - 执行setTimeout回调，输出 '2: setTimeout'
```

## 完整的事件循环图示

```javascript
// 更复杂的例子
console.log('1');

setTimeout(() => {
  console.log('2');
  Promise.resolve().then(() => {
    console.log('3');
  });
}, 0);

new Promise((resolve) => {
  console.log('4');
  resolve();
}).then(() => {
  console.log('5');
});

setTimeout(() => {
  console.log('6');
}, 0);

console.log('7');

// 输出顺序：
// 1, 4, 7, 5, 2, 3, 6

// 分析：
// 宏任务1（script）: 1, 4, 7 → 清空微任务: 5
// 宏任务2（setTimeout1）: 2 → 清空微任务: 3
// 宏任务3（setTimeout2）: 6
```

## async/await与事件循环

```javascript
// async/await本质是Promise的语法糖
async function async1() {
  console.log('async1 start');
  await async2();
  console.log('async1 end');
}

async function async2() {
  console.log('async2');
}

console.log('script start');

setTimeout(() => {
  console.log('setTimeout');
}, 0);

async1();

new Promise(resolve => {
  console.log('promise1');
  resolve();
}).then(() => {
  console.log('promise2');
});

console.log('script end');

// 输出顺序：
// script start
// async1 start
// async2
// promise1
// script end
// async1 end
// promise2
// setTimeout

// 关键点：await后面的代码相当于.then中的回调
```

## process.nextTick（Node.js特有）

```javascript
// Node.js中，process.nextTick的优先级高于Promise（属于微任务）
// 但实际上nextTick有自己的队列，在微任务之前执行

process.nextTick(() => {
  console.log('nextTick1');
});

Promise.resolve().then(() => {
  console.log('promise1');
});

process.nextTick(() => {
  console.log('nextTick2');
});

// Node.js输出：
// nextTick1
// nextTick2
// promise1

// nextTick队列优先于Promise微任务队列
```

## requestAnimationFrame的位置

```javascript
// requestAnimationFrame 既不是宏任务也不是微任务
// 它在浏览器渲染之前执行
// 执行顺序：...宏任务 → 微任务 → requestAnimationFrame → 渲染 → ...

button.addEventListener('click', () => {
  Promise.resolve().then(() => console.log('microtask'));
  requestAnimationFrame(() => console.log('rAF'));
  setTimeout(() => console.log('task'));
});

// 点击后输出：
// microtask
// rAF
// task
```

## 事件循环与浏览器渲染

```javascript
// 浏览器渲染时机：在一次事件循环中，当微任务队列清空后，渲染前

document.body.style.background = 'red';
document.body.style.background = 'blue';

// 上面两行代码不会渲染两次，因为渲染发生在事件循环的末尾
// 最终只会渲染蓝色

// Promise.resolve().then(() => {
//   document.body.style.background = 'green';
// });
// 微任务中的样式变化会在同一帧中渲染
```

## Node.js 的事件循环阶段

```
   ┌───────────────────────────┐
┌─>│           timers           │ ← setTimeout/setInterval 回调
│  └─────────────┬─────────────┘
│  ┌─────────────┴─────────────┐
│  │     pending callbacks     │ ← I/O回调延迟到下一轮
│  └─────────────┬─────────────┘
│  ┌─────────────┴─────────────┐
│  │       idle, prepare       │ ← 内部使用
│  └─────────────┬─────────────┘
│  ┌─────────────┴─────────────┐
│  │           poll            │ ← 轮询I/O事件
│  └─────────────┬─────────────┘
│  ┌─────────────┴─────────────┐
│  │           check           │ ← setImmediate 回调
│  └─────────────┬─────────────┘
│  ┌─────────────┴─────────────┐
│  │      close callbacks      │ ← 关闭事件
│  └───────────────────────────┘
```

## 常见面试题

```javascript
// 面试题1
Promise.resolve().then(() => {
  console.log(1);
  Promise.resolve().then(() => {
    console.log(2);
  });
});
Promise.resolve().then(() => {
  console.log(3);
});
// 输出：1, 3, 2

// 面试题2
setTimeout(() => {
  console.log(1);
}, 0);

new Promise(resolve => {
  console.log(2);
  resolve();
  console.log(3);
}).then(() => {
  console.log(4);
});

console.log(5);

// 输出：2, 3, 5, 4, 1

// 面试题3
setTimeout(() => {
  console.log('timeout1');
  Promise.resolve().then(() => console.log('promise1'));
}, 0);

setTimeout(() => {
  console.log('timeout2');
  Promise.resolve().then(() => console.log('promise2'));
}, 0);

// 输出：timeout1, promise1, timeout2, promise2
// 每个宏任务前都会清空微任务队列
```

## 宏任务与微任务的关系

```javascript
// 微任务可以"插队" — 在宏任务之间执行
setTimeout(() => console.log('task1'), 0);

Promise.resolve().then(() => {
  console.log('micro1');
  Promise.resolve().then(() => {
    console.log('micro2');
    Promise.resolve().then(() => {
      console.log('micro3');
    });
  });
});

setTimeout(() => console.log('task2'), 0);

// 输出：
// micro1, micro2, micro3, task1, task2

// 注意：微任务队列是在每个宏任务完成后清空
// 微任务中产生的新微任务也会在同一个周期被执行

// 如果微任务无限产生新微任务，会阻塞宏任务执行
Promise.resolve().then(function microTask() {
  console.log('micro');
  Promise.resolve().then(microTask); // 产生新微任务
});
setTimeout(() => console.log('task'), 0);
// 'micro' 会无限输出，task永远无法执行（主线程饥饿）
```
