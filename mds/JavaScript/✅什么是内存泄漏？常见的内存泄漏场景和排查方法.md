# ✅什么是内存泄漏？常见的内存泄漏场景和排查方法

# 典型回答

**内存泄漏（Memory Leak）** 是指程序申请的内存使用完后没有被正确释放，导致这部分内存无法被再次使用。随着时间推移，内存泄漏会导致可用内存越来越少，最终引发性能下降甚至程序崩溃。

**内存泄漏对应用的影响：**
- 页面响应越来越慢
- 浏览器占用内存持续增长
- 频繁触发垃圾回收导致卡顿
- 最终可能导致"Out of Memory"崩溃

**常见的内存泄漏场景：**

| 场景 | 原因 | 示例 |
|------|------|------|
| 意外的全局变量 | 未声明的变量或全局缓存 | `function() { leaked = 'value'; }` |
| 被遗忘的定时器 | setInterval未清除 | 定时器引用的对象无法释放 |
| 闭包 | 闭包长期持有外部变量 | React Hooks中的闭包陷阱 |
| DOM引用 | 移除DOM但JS仍持有引用 | `const el = document.getElementById('id')` |
| 事件监听器 | 未移除的事件监听 | addEventListener未remove |
| 脱离DOM的引用 | 从DOM树移除但未被GC | 变量中保存了DOM元素引用 |
| Map/Set中的键 | 对象键不被释放 | 使用WeakMap替代 |
| 大对象缓存 | 无限增长的缓存 | 缓存没有大小限制 |

# 扩展知识

## 意外的全局变量

```javascript
// 场景1：未声明的变量
function createLeak() {
  leaked = '全局变量'; // 严格模式会报错
  this.anotherLeak = '也泄露'; // this指向全局
}

// 场景2：全局数组无限增长
const cache = [];
function addToCache(item) {
  cache.push(item); // 不断增长，没有上限
}

// 解决方案
'use strict'; // 防止意外全局变量

// 限制缓存大小
class LimitedCache {
  constructor(maxSize = 100) {
    this.maxSize = maxSize;
    this.cache = new Map();
  }
  set(key, value) {
    if (this.cache.size >= this.maxSize) {
      const firstKey = this.cache.keys().next().value;
      this.cache.delete(firstKey);
    }
    this.cache.set(key, value);
  }
}
```

## 被遗忘的定时器

```javascript
// 内存泄漏示例
function startTimer() {
  const data = new Array(1000000).fill('heavy data');
  const timerId = setInterval(() => {
    console.log(data.length); // data一直被引用
  }, 1000);
  // 没有返回timerId，外部无法清除
}

// 正确写法
function startTimerSafe() {
  const data = new Array(1000000).fill('heavy data');
  const timerId = setInterval(() => {
    console.log(data.length);
  }, 1000);

  // 返回清除函数
  return () => clearInterval(timerId);
}

const stopTimer = startTimerSafe();
// 不再需要时
stopTimer();

// 请求动画帧同理
let animationId;
function startAnimation() {
  function animate() {
    // ...
    animationId = requestAnimationFrame(animate);
  }
  animate();
}
// 需要取消
// cancelAnimationFrame(animationId);
```

## DOM引用泄漏

```javascript
// 场景：保存DOM引用，但DOM被移除后仍被引用
class Dialog {
  constructor() {
    this.element = document.createElement('div');
    document.body.appendChild(this.element);
  }

  destroy() {
    document.body.removeChild(this.element);
    // this.element仍然引用着已移除的DOM
    // 如果Dialog实例长期存在，DOM不会被GC回收
  }

  // 正确做法
  destroyProperly() {
    document.body.removeChild(this.element);
    this.element = null; // 释放引用
  }
}

// 场景2：DOM元素引用在数组中
const elements = [];
function trackElement(el) {
  elements.push(el);
  // 即使DOM被移除，数组中的引用阻止GC

  // 更安全的做法：使用WeakRef
  const ref = new WeakRef(el);
}
```

## 事件监听器泄漏

```javascript
// 未移除的事件监听器
class Button {
  constructor(element) {
    this.element = element;
    this.handleClick = () => {
      console.log('clicked', this);
    };
    element.addEventListener('click', this.handleClick);
    // 没有removeEventListener
  }

  destroy() {
    this.element.removeEventListener('click', this.handleClick);
    this.element = null;
  }
}

// 被动引用问题
document.addEventListener('scroll', () => {
  // 匿名函数无法移除
});

// 正确的做法：使用具名函数或AbortController
const controller = new AbortController();
document.addEventListener('scroll', handler, { signal: controller.signal });
// 移除
controller.abort();
```

## React中的内存泄漏

```javascript
// React组件中的常见内存泄漏

// 场景1：组件卸载后setState
function UserProfile({ userId }) {
  const [user, setUser] = useState(null);

  useEffect(() => {
    fetchUser(userId).then(data => {
      // 如果组件在请求完成前卸载，setState会导致泄漏
      setUser(data);
    });
  }, [userId]);

  // 修复：
  useEffect(() => {
    let cancelled = false;
    fetchUser(userId).then(data => {
      if (!cancelled) setUser(data);
    });
    return () => { cancelled = true; };
  }, [userId]);
}

// 场景2：事件监听未清理
function Chat() {
  useEffect(() => {
    socket.on('message', handleMessage);
    return () => {
      socket.off('message', handleMessage); // 清理
    };
  }, []);
}

// 场景3：定时器未清理
function Timer() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    const id = setInterval(() => {
      setCount(c => c + 1);
    }, 1000);
    return () => clearInterval(id);
  }, []);
}
```

## 排查方法

```javascript
// 1. Chrome DevTools Performance面板
// - 录制性能，观察内存使用趋势
// - 检查JS Heap是否持续增长

// 2. Chrome DevTools Memory面板
// - 拍摄堆快照（Heap Snapshot）
// - 对比两个快照的差异
// - 查找"Detached DOM tree"（脱离DOM树的节点）

// 3. Allocation Timeline
// - 记录内存分配时间线
// - 定位大量持续分配的位置

// 4. Node.js内存分析
const v8 = require('v8');
const fs = require('fs');

// 生成堆快照
const snapshot = v8.getHeapSnapshot();
snapshot.pipe(fs.createWriteStream('heap.heapsnapshot'));

// 监控内存使用
function monitorMemory() {
  setInterval(() => {
    const usage = process.memoryUsage();
    if (usage.heapUsed > 500 * 1024 * 1024) { // 500MB
      console.warn('内存使用过高:', usage.heapUsed / 1024 / 1024, 'MB');
    }
  }, 10000);
}
```

## 检测工具的配置

```javascript
// Chrome DevTools
// 1. Performance → 勾选 Memory → 录制
// 2. Memory → Heap snapshot → Take snapshot
// 3. 查找 Detached HTMLDivElement（已删除但未释放的DOM）

// 在代码中添加性能标记
performance.mark('before-action');
// ... 执行操作 ...
performance.mark('after-action');
performance.measure('action-memory', 'before-action', 'after-action');

// 使用PerformanceObserver
const observer = new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    if (entry.name.includes('memory')) {
      console.log('内存事件:', entry);
    }
  }
});
observer.observe({ entryTypes: ['memory', 'measure'] });
```
