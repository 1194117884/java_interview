# ✅什么是受控组件和非受控组件？如何选择？

# 典型回答

**受控组件（Controlled Component）**和**非受控组件（Uncontrolled Component）**是React中管理表单输入的两种模式。

**受控组件**：表单元素的值由React的状态（state）控制。每当用户输入时，通过事件处理器更新state，state再通过value属性反馈给表单元素。数据的流动是单向闭环的：`用户输入 → 事件处理 → 更新state → 重新渲染 → 显示新值`。

```jsx
// 受控组件 —— state是唯一数据源
function ControlledInput() {
  const [value, setValue] = useState('');
  
  return (
    <input 
      value={value} 
      onChange={(e) => setValue(e.target.value)} 
    />
  );
}
```

**非受控组件**：表单元素的值由DOM自身管理，React不直接控制其值。通过ref获取DOM节点来读取当前值，而不是通过state实时同步。

```jsx
// 非受控组件 —— 由DOM管理值
function UncontrolledInput() {
  const inputRef = useRef(null);
  
  const handleSubmit = () => {
    console.log(inputRef.current.value);  // 使用时才读取
  };
  
  return <input ref={inputRef} defaultValue="initial" />;
}
```

**选择原则**：在大多数情况下，优先使用受控组件。受控组件提供了更强的数据控制能力和即时验证能力。只在某些特殊场景（如文件上传、集成非React代码、性能敏感的大表单）中考虑非受控组件。

# 扩展知识

### 两种模式的完整对比

| 特性 | 受控组件 | 非受控组件 |
|------|---------|-----------|
| 数据来源 | React state | DOM自身 |
| 实时验证 | 容易实现 | 较困难 |
| 即时条件禁用 | 容易（基于state） | 较复杂 |
| 动态输入格式 | 容易（如格式化输入） | 困难 |
| 文件上传 | 不支持 | 唯一方式 |
| 集成非React代码 | 困难 | 容易 |
| 性能（大表单） | 每次输入都渲染 | 无需实时渲染 |
| 代码量 | 稍多 | 较少 |
| 调试 | 容易（数据在state中） | 困难（数据在DOM中） |

### 各种表单元素的受控模式

```jsx
function FullControlledForm() {
  const [form, setForm] = useState({
    text: '',
    textarea: '',
    select: '',
    checkbox: false,
    radios: '',
    files: null,  // 文件只能非受控
  });
  
  const updateField = (name) => (e) => {
    const value = e.target.type === 'checkbox' 
      ? e.target.checked 
      : e.target.value;
    setForm(prev => ({ ...prev, [name]: value }));
  };
  
  return (
    <form>
      {/* 文本输入框 */}
      <input name="text" value={form.text} onChange={updateField('text')} />
      
      {/* 文本域 */}
      <textarea name="textarea" value={form.textarea} onChange={updateField('textarea')} />
      
      {/* 下拉选择框 */}
      <select name="select" value={form.select} onChange={updateField('select')}>
        <option value="">请选择</option>
        <option value="a">选项A</option>
        <option value="b">选项B</option>
      </select>
      
      {/* 复选框 */}
      <input type="checkbox" name="checkbox" checked={form.checkbox} onChange={updateField('checkbox')} />
      
      {/* 单选组 */}
      <input type="radio" name="radios" value="1" checked={form.radios === '1'} onChange={updateField('radios')} />
      <input type="radio" name="radios" value="2" checked={form.radios === '2'} onChange={updateField('radios')} />
    </form>
  );
}
```

### 受控组件的常见陷阱

```jsx
// 陷阱1：将undefined/null赋给value会导致输入框变为非受控
const [value, setValue] = useState(undefined);
return <input value={value} onChange={handleChange} />;
// 修复：初始化为空字符串
const [value, setValue] = useState('');

// 陷阱2：受控组件到非受控组件的转变
// 条件渲染时在不同模式间切换会触发React警告
const [mode, setMode] = useState('controlled'); // 'uncontrolled'
// ...会收到 "A component is changing from controlled to uncontrolled" 警告

// 陷阱3：数字输入的value处理
const [age, setAge] = useState('');
<input 
  value={age} 
  onChange={e => setAge(e.target.value)}  // e.target.value是字符串
/>
// 正确做法：按需转换
onChange={e => setAge(Number(e.target.value) || '')}
```

### 非受控组件的使用场景

```jsx
// 场景1：文件上传 —— 只能非受控
function FileUpload() {
  const fileRef = useRef(null);
  
  const handleSubmit = async () => {
    const file = fileRef.current.files[0];
    const formData = new FormData();
    formData.append('file', file);
    await axios.post('/upload', formData);
  };
  
  return (
    <>
      <input type="file" ref={fileRef} />
      <button onClick={handleSubmit}>上传</button>
    </>
  );
}

// 场景2：集成jQuery插件等非React代码
function DatePickerWrapper() {
  const containerRef = useRef(null);
  
  useEffect(() => {
    // jQuery插件操作DOM，不适合受控模式
    $(containerRef.current).datepicker({
      onChange: (date) => console.log(date)
    });
  }, []);
  
  return <div ref={containerRef} />;
}

// 场景3：大表单的性能优化
// 当一个表单有大量输入且不需要即时验证时
// 使用非受控组件避免每次按键都触发React渲染
```

### React官方推荐

React官方文档的明确建议：
- **大多数场景使用受控组件**：更符合React的数据流模式，更可预测
- **非受控组件作为逃生舱**：在需要与非React代码集成时使用
- **避免在受控和非受控间切换**：一旦确定模式就不要在运行时切换
