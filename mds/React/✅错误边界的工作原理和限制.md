# ✅错误边界的工作原理和限制

# 典型回答

错误边界（Error Boundary）是React提供的一种**捕获子组件树中JavaScript错误**的组件，它可以防止整个React应用崩溃，并在捕获错误后显示备用UI。

错误边界是一个类组件，实现了 `static getDerivedStateFromError()` 和/或 `componentDidCatch()` 生命周期方法：

```jsx
class ErrorBoundary extends React.Component {
  state = { hasError: false, error: null };
  
  // 渲染阶段调用：更新state使下次渲染显示fallback UI
  static getDerivedStateFromError(error) {
    return { hasError: true, error };
  }
  
  // 提交阶段调用：用于副作用（日志记录、错误上报）
  componentDidCatch(error, errorInfo) {
    logErrorToService(error, errorInfo.componentStack);
  }
  
  render() {
    if (this.state.hasError) {
      return this.props.fallback || <h1>Something went wrong.</h1>;
    }
    return this.props.children;
  }
}

// 使用
<ErrorBoundary fallback={<div>页面出错了</div>}>
  <MyComponent />
</ErrorBoundary>
```

**错误边界的关键限制**：错误边界**只能捕获**以下类型的错误：
- 渲染期间（render阶段）抛出的错误
- 生命周期方法中抛出的错误
- 构造函数中的错误

**错误边界无法捕获**的错误：
- 事件处理器中的错误
- 异步代码中的错误（setTimeout、Promise、requestAnimationFrame）
- 服务端渲染的错误
- 错误边界自身抛出的错误

# 扩展知识

### 为什么错误边界用类组件？

React官方选择类组件实现错误边界的原因在于：**目前只有类组件提供了 `componentDidCatch` 和 `getDerivedStateFromError` 这两个生命周期方法**。函数组件中虽然有 `useEffect`，但没有对应的错误捕获Hook。

```jsx
// 函数组件目前无法实现错误边界
function ErrorBoundary({ children }) {
  // ❌ 函数组件中无法实现 componentDidCatch
  // ❌ 没有对应的Hook（React团队正在考虑）
}
```

### 嵌套错误边界的粒度控制

```jsx
function App() {
  return (
    <ErrorBoundary fallback={<div>整体页面崩溃</div>}>
      <Header />
      
      <ErrorBoundary fallback={<div>主要内容区出错了</div>}>
        <main>
          <ErrorBoundary fallback={<div>侧边栏组件出错</div>}>
            <Sidebar />
          </ErrorBoundary>
          <ErrorBoundary fallback={<div>主内容组件出错</div>}>
            <Content />
          </ErrorBoundary>
        </main>
      </ErrorBoundary>
      
      <Footer />
    </ErrorBoundary>
  );
}
// 粒度控制的意义：
// - Sidebar出错 → 只Sidebar显示fallback，Content不受影响
// - Content出错 → 只Content显示fallback，Sidebar继续工作
// - main整体出错 → 显示"主要内容区出错了"fallback
```

### 错误恢复尝试

```jsx
// 带重试功能的错误边界
class ErrorBoundaryWithRetry extends React.Component {
  state = { hasError: false, error: null, errorInfo: null };
  
  static getDerivedStateFromError(error) {
    return { hasError: true, error };
  }
  
  componentDidCatch(error, errorInfo) {
    this.setState({ errorInfo });
    // 上报错误
    reportError(error, errorInfo);
  }
  
  handleRetry = () => {
    this.setState({ hasError: false, error: null, errorInfo: null });
  };
  
  render() {
    if (this.state.hasError) {
      return (
        <div className="error-boundary-fallback">
          <h2>组件加载出错</h2>
          <details>
            <summary>错误详情</summary>
            <pre>{this.state.error?.toString()}</pre>
            <pre>{this.state.errorInfo?.componentStack}</pre>
          </details>
          <button onClick={this.handleRetry}>重试</button>
        </div>
      );
    }
    return this.props.children;
  }
}
```

### 事件处理器中错误的处理

```jsx
// ❌ 错误边界无法捕获事件处理器中的错误
function BrokenButton() {
  const handleClick = () => {
    throw new Error('Button click error');  // 不会被错误边界捕获！
    // 事件处理器错误需要自行try-catch
  };
  
  return <button onClick={handleClick}>Click me</button>;
}

// ✅ 事件处理器中需要自己处理错误
function SafeButton({ onClick }) {
  const handleClick = async (e) => {
    try {
      await onClick(e);
    } catch (error) {
      console.error('Button error:', error);
      // 可以选择将错误抛给错误边界
      // 但error boundary无法直接捕获事件处理器错误
      // 可以使用state触发重新渲染来让错误边界捕获
    }
  };
  
  return <button onClick={handleClick}>Safe Click</button>;
}

// ✅ 包装函数捕获事件错误
function useErrorHandler() {
  const [error, setError] = useState(null);
  
  if (error) {
    throw error;  // 将错误抛给错误边界
  }
  
  return setError;
}

function AsyncComponent() {
  const handleError = useErrorHandler();
  
  const handleClick = async () => {
    try {
      await riskyOperation();
    } catch (error) {
      handleError(error);  // 通过throw让错误边界捕获
    }
  };
  
  return <button onClick={handleClick}>Risky Click</button>;
}
```

### 异步错误的处理

```jsx
// ❌ 异步错误不能被错误边界自动捕获
function AsyncBroken() {
  useEffect(() => {
    // setTimeout中的错误
    setTimeout(() => {
      throw new Error('Async error');  // 不会被捕获！
    }, 1000);
    
    // Promise中的错误
    fetch('/api/data')
      .then(response => {
        throw new Error('Fetch error');  // 不会被捕获！
      });
  }, []);
  
  return <div>Content</div>;
}

// ✅ 正确处理异步错误
function AsyncSafe() {
  const [data, setData] = useState(null);
  const [error, setError] = useState(null);
  
  useEffect(() => {
    fetch('/api/data')
      .then(response => {
        if (!response.ok) throw new Error('Network error');
        return response.json();
      })
      .then(setData)
      .catch(setError);  // 捕获异步错误到state
  }, []);
  
  // 通过state抛出错误，让错误边界捕获
  if (error) throw error;
  
  if (!data) return <div>Loading...</div>;
  
  return <div>{data.name}</div>;
}
```

### 错误边界的完整对比表

| 错误类型 | 是否被错误边界捕获 | 处理方式 |
|---------|-----------------|---------|
| render阶段错误 | 是 | 自动捕获，显示fallback |
| componentDidMount错误 | 是 | 自动捕获 |
| componentDidUpdate错误 | 是 | 自动捕获 |
| 构造函数错误 | 是 | 自动捕获 |
| 事件处理器错误 | 否 | try-catch或useErrorHandler |
| setTimeout/requestAnimationFrame | 否 | 自行try-catch |
| Promise/async/await | 否 | .catch或try-catch |
| 服务端渲染错误 | 否 | 服务端错误处理 |
| 错误边界自身错误 | 否 | 上层错误边界捕获 |

### 生产环境和开发环境的行为差异

```jsx
// 生产环境下，错误边界捕获的错误会被静默处理
// 但console.error仍然会输出

// 开发模式下，React会额外显示完整的错误信息和组件栈
// 帮助开发者定位问题

// 建议的开发环境配置
if (process.env.NODE_ENV === 'development') {
  // 在开发环境下显示更详细的错误信息
  ErrorBoundary.prototype.componentDidCatch = function(error, info) {
    console.group('Error Boundary Caught');
    console.error('Error:', error);
    console.error('Component Stack:', info.componentStack);
    console.groupEnd();
  };
}
```

### 错误边界与Suspense的区别

```jsx
// 错误边界 → 捕获错误，显示fallback
<ErrorBoundary fallback={<ErrorUI />}>
  <Component />
</ErrorBoundary>

// Suspense → 捕获Promise（加载状态），显示fallback
<Suspense fallback={<Loading />}>
  <LazyComponent />
</Suspense>

// 可以组合使用
<ErrorBoundary fallback={<ErrorUI />}>
  <Suspense fallback={<Loading />}>
    <LazyComponent />
  </Suspense>
</ErrorBoundary>
// 先显示Loading → 加载完成显示组件
// 如果加载失败 → ErrorBoundary捕获显示ErrorUI
```
