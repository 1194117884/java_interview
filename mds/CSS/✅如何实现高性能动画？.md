# ✅如何实现高性能动画？

# 典型回答

实现高性能 CSS 动画的核心原则是：**只触发合成（composite），避免触发重排（reflow）和重绘（repaint）**。

## 浏览器渲染管线

浏览器渲染一帧画面需要经过以下步骤：

```
JavaScript → 样式计算 → 布局（Layout） → 绘制（Paint） → 合成（Composite）
```

每一步都会消耗性能，动画性能优化的目标就是尽量减少经过的步骤：

| 触发操作 | 管线路径 | 性能 |
|---------|---------|------|
| **重排（Reflow）** | 布局 → 绘制 → 合成 | 最差 |
| **重绘（Repaint）** | 绘制 → 合成 | 中等 |
| **合成（Composite）** | 仅合成 | 最佳（60fps） |

## 高性能动画的核心原则

### 原则 1：只使用 transform 和 opacity 做动画

```css
/* 高性能：仅触发合成 */
.best {
  transform: translateX(100px);
  opacity: 0.5;
}

/* 低性能：触发重排 */
.bad {
  left: 100px;           /* 触发布局 */
  width: 200px;          /* 触发布局 */
  margin-left: 50px;     /* 触发布局 */
}

/* 中等性能：触发重绘 */
.ok {
  color: red;            /* 触发重绘 */
  background-color: blue;/* 触发重绘 */
}
```

### 原则 2：利用 GPU 加速

```css
/* 强制启用 GPU 合成层 */
.gpu-layer {
  transform: translateZ(0);  /* 触发 GPU 合成 */
  /* 或 */
  will-change: transform;     /* 提前告知浏览器 */
}
```

### 原则 3：减少动画元素的层级和复杂度

```css
/* 避免动画元素的 box-shadow 变化 */
.avoid-this {
  transition: box-shadow 0.3s;  /* box-shadow 动画性能开销大 */
}

/* 替代方案 */
.better {
  transition: opacity 0.3s;
  /* 用伪元素模拟阴影，只对 opacity 做动画 */
}
```

# 扩展知识

## 触发重排的常见属性

以下属性的修改会触发**重排（Layout）**，应避免在动画中使用：

| 分类 | 属性 |
|------|------|
| **盒模型** | `width`、`height`、`padding`、`margin`、`border` |
| **定位** | `top`、`left`、`right`、`bottom` |
| **布局** | `display`、`float`、`position`、`overflow` |
| **文本** | `font-size`、`font-weight`、`line-height`、`text-align` |
| **Flex/Grid** | `flex`、`grid-template-*`、`gap` |

## 触发重绘的常见属性

以下属性的修改会触发**重绘（Paint）**，性能开销中等：

| 分类 | 属性 |
|------|------|
| **颜色** | `color`、`background-color`、`border-color` |
| **背景** | `background-image`、`background-gradient` |
| **轮廓** | `outline`、`outline-color` |
| **可见性** | `visibility`（不影响布局） |

## 仅触发合成的属性

以下属性的修改只触发**合成（Composite）**，性能最佳：

| 属性 | 说明 |
|------|------|
| `transform` | 平移、旋转、缩放、倾斜 |
| `opacity` | 透明度变化 |

## 图层提升策略

现代浏览器使用**图层（Layer）** 机制来优化渲染。将动画元素提升到独立图层，可以避免影响其他元素：

### 触发图层提升的方式

```css
/* 1. will-change 属性（推荐） */
.animated {
  will-change: transform, opacity;
}

/* 2. translateZ(0) 或 translate3d(0,0,0) 技巧 */
.animated {
  transform: translateZ(0);
}

/* 3. 3D 变换自动触发 */
.animated {
  transform: rotateX(0);
}
```

### will-change 的注意事项

```css
/* 正确用法：动画前设置，动画结束后移除 */
.element {
  will-change: transform;
  transition: transform 0.3s;
}

.element:hover {
  transform: scale(1.1);
}

/* 错误用法：不要滥用，会消耗大量内存 */
.everything {
  will-change: transform, opacity, left, top;  /* 太多！ */
}
```

**最佳实践**：

```css
/* 在 JS 中添加，动画结束后移除 */
element.style.willChange = 'transform';
// 执行动画
element.style.transform = 'translateX(100px)';
// 动画完成后
element.addEventListener('transitionend', () => {
  element.style.willChange = 'auto';
});
```

## requestAnimationFrame 的使用

对于 JS 驱动的动画，使用 `requestAnimationFrame` 而非 `setTimeout`：

```javascript
// 不推荐
function animateBad() {
  element.style.left = x + 'px';
  setTimeout(animateBad, 16); // 不精确的 60fps
}

// 推荐
function animateGood(timestamp) {
  element.style.transform = `translateX(${x}px)`;
  requestAnimationFrame(animateGood);
}
requestAnimationFrame(animateGood);
```

## 性能对比测试表

| 动画方式 | 触发操作 | 可达到帧率 | 内存消耗 | 推荐度 |
|---------|---------|-----------|---------|-------|
| `transform` + `opacity` | 仅合成 | 60fps | 低 | 最优 |
| `transition` + `transform` | 仅合成 | 60fps | 低 | 推荐 |
| `animation` + `transform` | 仅合成 | 60fps | 中 | 推荐 |
| 修改 `left`/`top` | 重排 | 30-60fps | 低 | 不推荐 |
| 修改 `width`/`height` | 重排 | 20-40fps | 低 | 不推荐 |
| 修改 `box-shadow` | 重绘 | 20-40fps | 中 | 不推荐 |
| `requestAnimationFrame` | 取决于操作 | 60fps | 中 | 适合 JS 动画 |
| Web Animations API | 合成 | 60fps | 低 | 现代方案 |
| Canvas/WebGL | 取决于操作 | 60fps | 高 | 适合复杂动画 |

## 其他性能优化技巧

### 1. 使用 contain 属性

```css
.animated {
  contain: layout style paint;
  /* 告诉浏览器该元素内部变化不影响到外部 */
}
```

### 2. 减少 paint 区域

```css
/* 移除不必要的阴影动画 */
.card {
  /* 避免 */
  transition: box-shadow 0.3s;
  box-shadow: 0 2px 8px rgba(0,0,0,.15);
}

/* 改用伪元素 */
.card::before {
  content: '';
  position: absolute;
  inset: 0;
  opacity: 0;
  box-shadow: 0 8px 24px rgba(0,0,0,.15);
  transition: opacity 0.3s;
}
.card:hover::before {
  opacity: 1;
}
```

### 3. 使用硬件加速

```css
/* 移动端特别推荐 */
.mobile-animated {
  -webkit-transform: translateZ(0);
  transform: translateZ(0);
}
```

### 4. 减少动画元素数量

```css
/* 应用虚拟化或懒加载，减少页面上同时动画的元素 */
```

### 5. 使用 will-change 做预优化

```css
/* 在动画开始前告知浏览器 */
.changing {
  will-change: transform;
}
```

## DevTools 性能分析

使用 Chrome DevTools 的 Performance 面板分析动画性能：

1. **FPS 计数器**：查看实时帧率
2. **Layers 面板**：查看图层分配
3. **Paint flashing**：高亮重绘区域
4. **Rendering 设置**：开启 `Layer borders` 查看合成层

```bash
# Chrome DevTools 快捷键
Cmd/Ctrl + Shift + P → 输入 "fps" → 显示 FPS 计数器
Cmd/Ctrl + Shift + P → 输入 "paint" → 显示重绘区域
```

## 动画性能检测工具

```javascript
// 简单 FPS 监测
let lastTime = performance.now();
let frames = 0;

function checkFPS(now) {
  frames++;
  if (now - lastTime >= 1000) {
    console.log(`FPS: ${frames}`);
    frames = 0;
    lastTime = now;
  }
  requestAnimationFrame(checkFPS);
}
requestAnimationFrame(checkFPS);
```

## 总结：性能优化检查清单

| 检查项 | 说明 |
|-------|------|
| 只对 `transform` 和 `opacity` 做动画 | 避免重排 |
| 动画元素提升为独立图层 | 使用 `will-change` |
| 减少同时动画的元素数量 | 每个独立图层消耗内存 |
| 避免动画导致大面积重绘 | 使用 DevTools 检查 |
| 移动端注意内存限制 | 不要过度使用 GPU 加速 |
| 使用 `requestAnimationFrame` | 避免 JS 动画丢帧 |
| 配合 `contain` 属性 | 减少样式计算范围 |
