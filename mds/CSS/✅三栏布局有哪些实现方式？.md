# ✅三栏布局有哪些实现方式？

# 典型回答

三栏布局（圣杯布局/Holy Grail Layout）是一种经典的页面布局模式：**左右两栏固定宽度，中间栏自适应**。

以下是常见实现方式及其对比：

| 实现方式 | 优点 | 缺点 | 兼容性 |
|---------|------|------|--------|
| Flexbox 方案 | 代码简洁，语义好 | 内容顺序依赖 HTML 结构 | 全部现代浏览器 |
| Grid 方案 | 最简洁，不需额外结构调整 | 不支持 IE | 现代浏览器 |
| 浮动 + 负 margin | 兼容性好 | 代码复杂，需注意清除浮动 | IE6+ |
| 绝对定位 | 实现简单 | 脱离文档流，需手动处理高度 | 全部 |
| table 布局 | 高度自动等高 | 不符合语义化 | 全部 |
| 圣杯布局（浮动版） | 经典方案，中间栏优先渲染 | 实现复杂，需负 margin 技巧 | IE6+ |
| 双飞翼布局 | 圣杯的改进版，更简洁 | 需要额外 inner 容器 | IE6+ |

## Flexbox 实现（推荐方案）

```html
<div class="layout">
  <aside class="left">Left Sidebar</aside>
  <main class="center">Main Content</main>
  <aside class="right">Right Sidebar</aside>
</div>
```

```css
.layout {
  display: flex;
  min-height: 100vh;
}

.left {
  width: 200px;         /* 固定宽度 */
  flex-shrink: 0;       /* 禁止缩小 */
}

.center {
  flex: 1;              /* 自动占满剩余空间 */
}

.right {
  width: 200px;
  flex-shrink: 0;
}
```

**中间栏优先渲染**（通过 order 属性）：

```html
<div class="layout">
  <main class="center">Main Content</main>   <!-- 放在前面优先渲染 -->
  <aside class="left">Left</aside>
  <aside class="right">Right</aside>
</div>
```

```css
.center { flex: 1; order: 2; }
.left   { width: 200px; flex-shrink: 0; order: 1; }
.right  { width: 200px; flex-shrink: 0; order: 3; }
```

## Grid 实现（最简洁）

```css
.layout {
  display: grid;
  grid-template-columns: 200px 1fr 200px;  /* 左右固定，中间自适应 */
  grid-template-rows: auto 1fr auto;        /* 如果是完整页布局 */
  min-height: 100vh;
  gap: 0;
}
```

```html
<div class="layout">
  <aside class="left">Left</aside>
  <main class="center">Center</main>
  <aside class="right">Right</aside>
</div>
```

或者使用命名的 Grid 区域：

```css
.layout {
  display: grid;
  grid-template-columns: 200px 1fr 200px;
  grid-template-areas: "left center right";
}

.left   { grid-area: left; }
.center { grid-area: center; }
.right  { grid-area: right; }
```

# 扩展知识

## 圣杯布局（经典浮动实现）

**特点**：中间栏优先渲染，通过 padding 和负 margin 实现

```html
<div class="container">
  <main class="center">中间栏（优先渲染）</main>
  <aside class="left">左栏</aside>
  <aside class="right">右栏</aside>
</div>
```

```css
.container {
  padding: 0 200px 0 200px;  /* 为左右两栏留出空间 */
  overflow: hidden;          /* 清除浮动 */
}

.center {
  float: left;
  width: 100%;               /* 占满容器 */
}

.left {
  float: left;
  width: 200px;
  margin-left: -100%;        /* 拉到容器最左边 */
  position: relative;
  left: -200px;              /* 再向左移回自己的位置 */
}

.right {
  float: left;
  width: 200px;
  margin-left: -200px;       /* 拉到容器最右边 */
  position: relative;
  right: -200px;             /* 向右移回自己的位置 */
}
```

## 双飞翼布局

**特点**：圣杯布局的改进，用额外的 inner 容器代替 padding

```html
<div class="container">
  <main class="center">
    <div class="center-inner">中间栏</div>
  </main>
  <aside class="left">左栏</aside>
  <aside class="right">右栏</aside>
</div>
```

```css
.center {
  float: left;
  width: 100%;
}

.center-inner {
  margin: 0 200px;           /* 用 margin 给左右栏留空间 */
}

.left {
  float: left;
  width: 200px;
  margin-left: -100%;        /* 回到最左边 */
}

.right {
  float: left;
  width: 200px;
  margin-left: -200px;        /* 回到最右边 */
}

.container {
  overflow: hidden;          /* 清除浮动 */
}
```

## 浮动方案（简单版）

**特点**：实现简单，但中间栏不能优先渲染

```html
<div class="layout">
  <aside class="left">左栏</aside>
  <aside class="right">右栏</aside>
  <main class="center">中间栏</main>
</div>
```

```css
.left {
  float: left;
  width: 200px;
}

.right {
  float: right;
  width: 200px;
}

.center {
  /* 自动填充中间区域 */
  margin: 0 200px;           /* 给左右栏留空间 */
}

.layout {
  overflow: hidden;          /* 清除浮动 */
}
```

## 绝对定位方案

**特点**：实现简单，但需要手动处理高度

```css
.layout {
  position: relative;
  min-height: 300px;
}

.left {
  position: absolute;
  left: 0;
  top: 0;
  width: 200px;
  height: 100%;
}

.right {
  position: absolute;
  right: 0;
  top: 0;
  width: 200px;
  height: 100%;
}

.center {
  margin: 0 200px;           /* 留出左右空间 */
}
```

## Table 布局方案

**特点**：兼容性好，自动等高

```css
.layout {
  display: table;
  width: 100%;
  min-height: 300px;
}

.layout > * {
  display: table-cell;
}

.left, .right {
  width: 200px;
}

.center {
  /* 自动占满剩余宽度 */
}
```

## 各种方案的对比总结

| 方案 | 代码量 | 中间栏优先渲染 | 高度自适应 | 兼容性 | 推荐度 |
|------|-------|---------------|-----------|-------|-------|
| Flexbox | 少 | 支持（order） | 好 | 现代浏览器 | 极推荐 |
| Grid | 最少 | 支持（order） | 好 | 现代浏览器 | 极推荐 |
| 浮动 + 负 margin（圣杯） | 多 | 支持 | 好 | IE6+ | 兼容老项目 |
| 双飞翼 | 中 | 支持 | 好 | IE6+ | 兼容老项目 |
| 简单浮动 | 少 | 不支持 | 好 | IE6+ | 简单场景 |
| 绝对定位 | 少 | 支持 | 需手动 | 全部 | 简单场景 |
| Table | 少 | 不支持 | 自动等高 | 全部 | 不推荐 |

## 响应式适配

以上方案增加媒体查询即可适配移动端：

```css
@media (max-width: 768px) {
  .layout {
    flex-direction: column;     /* Flexbox 方案 */
    /* 或 */
    grid-template-columns: 1fr; /* Grid 方案 */
  }

  .left, .right {
    width: 100%;
  }
}
```

## 现代推荐

对于现代项目，**推荐优先使用 Grid** 实现三栏布局，因为 Grid 是真正的二维布局系统，代码最简洁直观。如果考虑 Flexbox 的一维特性更符合"三栏即一行"的语义，也可以选择 Flexbox。

```css
/* 最推荐：Grid 方案，一行代码解决 */
.layout {
  display: grid;
  grid-template-columns: 200px 1fr 200px;
  gap: 24px;
}
```
