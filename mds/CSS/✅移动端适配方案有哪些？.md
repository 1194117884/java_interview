# ✅移动端适配方案有哪些？

# 典型回答

移动端适配的核心目标是：**在不同屏幕尺寸和分辨率的设备上，页面内容都能合理显示**。常见的适配方案有以下几种：

## 主流适配方案对比

| 方案 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|---------|
| **rem 适配** | 根据屏幕宽度动态设置 html 的 font-size，元素使用 rem 单位 | 兼容性好，易于维护 | 需 JS 动态计算，小数像素问题 | 普通 H5 页面 |
| **vw/vh 适配** | 使用视口单位，100vw = 视口宽度 | 纯 CSS，无需 JS | 不能直接限制最大宽度 | 简单页面 |
| **rem + vw 结合** | 用 vw 设置 html 的 font-size，用 rem 设置元素尺寸 | 纯 CSS，结合两者优点 | 需要计算比例 | 推荐方案 |
| **媒体查询** | 通过 @media 为不同屏幕宽度设置不同样式 | 精准控制断点 | 工作量大，维护成本高 | 响应式网站 |
| **百分比布局** | 使用百分比实现流式布局 | 简单直观 | 嵌套时计算复杂 | 简单页面 |
| **flexible 方案** | 阿里手淘方案，动态 rem | 成熟稳定 | 需引入 JS | 淘宝系 H5 |
| **viewport 缩放** | 通过 viewport 的 initial-scale 控制 | 简单粗暴 | 文字可能模糊 | 非主流 |

## 推荐方案：rem + vw 适配

```css
/* 以 375px 设计稿为基准 */
/* 1rem = 100px（设计稿） */
/* 100vw = 375px → 1vw = 3.75px */
/* 100px = 100 / 3.75 = 26.6667vw */

html {
  font-size: 26.6667vw;  /* 375px 视口下 font-size = 100px */
}

/* 限制最大宽度 */
@media (min-width: 750px) {
  html {
    font-size: 53.3334vw; /* 750px 下保持比例 */
  }
}
```

```css
/* 设计稿宽度 375px，元素宽度 100px */
.box {
  width: 1rem;          /* 375px 下 = 100px */
  /* 在 414px 下自动放大到 ≈ 110px */
}
```

## 媒体查询适配

```css
/* 断点策略 */
/* 手机竖屏 */
@media (max-width: 575px) { ... }

/* 手机横屏 / 小平板 */
@media (min-width: 576px) and (max-width: 767px) { ... }

/* 平板 */
@media (min-width: 768px) and (max-width: 991px) { ... }

/* 小桌面 */
@media (min-width: 992px) and (max-width: 1199px) { ... }

/* 大桌面 */
@media (min-width: 1200px) { ... }
```

# 扩展知识

## 阿里 flexible 方案详解

手淘 flexible 方案的核心逻辑：

```javascript
// 简化版 flexible 实现
(function flexible() {
  const docEl = document.documentElement;
  const dpr = window.devicePixelRatio || 1;

  // 设置 data-dpr 属性
  docEl.setAttribute('data-dpr', dpr);

  function setRemUnit() {
    const width = docEl.clientWidth;
    // 将屏幕宽度分为 10 份，每份为 1rem
    const rem = width / 10;
    docEl.style.fontSize = rem + 'px';
  }

  setRemUnit();
  window.addEventListener('resize', setRemUnit);
})();
```

使用 flexible 时，设计稿通常以 750px 宽度出图，通过 PostCSS 插件（`postcss-pxtorem`）自动将 px 转换为 rem：

```javascript
// postcss.config.js
module.exports = {
  plugins: {
    'postcss-pxtorem': {
      rootValue: 75,     // 750 设计稿，1rem = 75px
      propList: ['*'],
    }
  }
};
```

```css
/* 源码 */
.box {
  width: 100px;
  height: 50px;
  font-size: 14px;
}

/* 编译后 */
.box {
  width: 1.3333rem;
  height: 0.6667rem;
  font-size: 0.1867rem;
}
```

## Viewport 适配方案

通过设置 viewport 参数实现适配：

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

核心参数说明：

| 参数 | 说明 |
|------|------|
| `width=device-width` | 页面宽度 = 设备宽度（物理 CSS 像素） |
| `initial-scale=1.0` | 初始缩放比例为 1 |
| `minimum-scale=1.0` | 最小缩放比例 |
| `maximum-scale=1.0` | 最大缩放比例 |
| `user-scalable=no` | 禁止用户缩放 |

## 高清屏适配（Retina 屏幕）

在高清屏（2x、3x）上，普通 1px 边框看起来过粗，需要做特殊处理：

```css
/* 使用 0.5px 边框（部分设备支持） */
.border-bottom {
  border-bottom: 0.5px solid #ddd;
}

/* 使用 transform 缩放 */
.border-1px {
  position: relative;
}
.border-1px::after {
  content: "";
  position: absolute;
  bottom: 0;
  left: 0;
  width: 100%;
  height: 1px;
  background: #ddd;
  transform: scaleY(0.5);  /* 在 2x 屏上变为 0.5px 物理效果 */
  transform-origin: 0 0;
}

/* 使用阴影模拟 */
.border-1px {
  box-shadow: inset 0 -1px 0 0 #ddd;
}
```

## 图片适配

不同分辨率设备加载不同尺寸的图片：

```html
<!-- 使用 srcset 和 sizes -->
<img
  src="photo-400.jpg"
  srcset="photo-400.jpg 400w, photo-800.jpg 800w, photo-1200.jpg 1200w"
  sizes="(max-width: 600px) 100vw, (max-width: 1200px) 50vw, 33vw"
  alt="photo"
>

<!-- 使用 picture 标签 -->
<picture>
  <source media="(min-resolution: 2dppx)" srcset="photo@2x.jpg">
  <source media="(min-width: 768px)" srcset="photo-desktop.jpg">
  <img src="photo.jpg" alt="photo">
</picture>
```

## 适配策略的选择

### 按项目类型选择

| 项目类型 | 推荐方案 | 原因 |
|---------|---------|------|
| **内容型网站**（新闻、博客） | 响应式 + 媒体查询 | 内容阅读体验优先，不同设备展示不同布局 |
| **功能型 H5**（表单、工具） | rem/vw 适配 | 需精确控制元素尺寸 |
| **混合开发 / WebView** | rem/vw + flexible | 与 Native 统一设计规范 |
| **后台管理系统** | 响应式 | 主要在桌面端使用，适配平板即可 |

### 设计稿尺寸选择

| 设计稿宽度 | 适用场景 |
|-----------|---------|
| 375px | iPhone 6/7/8/X 逻辑分辨率，现代主流 |
| 414px | iPhone Plus 系列 |
| 750px | 2x 图设计稿，flexible 方案常用 |
| 1125px | 3x 图设计稿 |

## 常见问题处理

### iOS Safari 底部安全区

```css
/* iPhone X 及以上机型适配 */
.safe-bottom {
  padding-bottom: constant(safe-area-inset-bottom);  /* iOS 11.0 */
  padding-bottom: env(safe-area-inset-bottom);        /* iOS 11.2+ */
}
```

### 1px 边框 Retina 适配

```css
/* PostCSS 插件自动处理 */
/* postcss-write-svg 或 postcss-px-to-viewport 等 */

/* 手动处理 */
@media (-webkit-min-device-pixel-ratio: 2), (min-resolution: 192dpi) {
  .border-1px {
    border-width: 0.5px;
  }
}
```

### 横竖屏适配

```css
/* 横屏检测 */
@media (orientation: landscape) {
  .header { height: 50px; }
}

/* 竖屏检测 */
@media (orientation: portrait) {
  .header { height: 80px; }
}
```

## 现代 CSS 新方案

对于只支持现代浏览器的项目，可以使用容器查询（Container Queries）：

```css
.component {
  container-type: inline-size;
}

@container (max-width: 400px) {
  .component .title {
    font-size: 1rem;
  }
}
```

这是未来的方向，可以让组件根据自身容器大小而非视口来响应适配。
