# ✅CSS选择器的优先级是如何计算的？

# 典型回答

CSS选择器的优先级（Specificity）是浏览器用来决定当多个规则冲突时，哪一条规则生效的计算机制。优先级用一个四位数字（或三部分组成的权重值）来表示，数值越高，优先级越高。

优先级计算规则如下：

1. **内联样式**：优先级最高，权重记为 `1,0,0,0`。
2. **ID选择器**（如 `#header`）：权重记为 `0,1,0,0`。
3. **类选择器、属性选择器、伪类**（如 `.class`、`[type="text"]`、`:hover`）：权重记为 `0,0,1,0`。
4. **元素选择器、伪元素**（如 `div`、`::before`）：权重记为 `0,0,0,1`。
5. **通配选择器**（`*`）、**组合器**（`+`、`>`、`~`、空格）、**否定伪类**（`:not()`）：不贡献优先级。但 `:not()` 内部的参数选择器会正常计入优先级。

## 优先级比较示例

```css
/* 优先级: 0,0,0,1 */
div { color: red; }

/* 优先级: 0,0,1,0 */
.class { color: blue; }

/* 优先级: 0,1,0,0 */
#id { color: green; }

/* 优先级: 0,1,0,1 */
div#id { color: yellow; }

/* 优先级: 0,1,1,0 */
#id.class { color: purple; }

/* 优先级: 0,0,2,1 */
div.class1.class2 { color: orange; }
```

如果优先级相同，则**后出现的规则覆盖先出现的规则**（就近原则）。

# 扩展知识

## `!important` 与优先级的关系

`!important` 是一个特殊标记，它可以**覆盖**任何普通声明的优先级。当 `!important` 出现时，它实际上脱离常规的优先级计算体系，形成单独的层级：

1. 浏览器默认样式
2. 用户普通声明
3. 作者普通声明
4. 作者 `!important` 声明
5. 用户 `!important` 声明
6. 动画样式
7. 过渡样式

当多个 `!important` 冲突时，仍按常规优先级规则比较。

```css
#id {
  color: red !important; /* 优先级极高 */
}

.class {
  color: blue !important; /* 同样有 !important，但优先级低于上面的 */
}
```

**最佳实践**：尽量避免使用 `!important`，它会让调试变得困难。应该通过合理的优先级设计来避免冲突，而不是依赖 `!important` 覆盖。

## 优先级计算的细节规则

### `:is()` 和 `:where()` 的特殊行为

这两个现代伪类函数的优先级计算规则有所不同：

- **`:is()`**：优先级取参数列表中**最高**的那个选择器的优先级
- **`:where()`**：优先级始终为 `0,0,0,0`，不贡献任何权重

```css
/* 优先级: 0,1,0,0 — 取 #id 的权重 */
:is(#id, .class, div) { color: red; }

/* 优先级: 0,0,0,0 — :where() 不贡献优先级 */
:where(#id, .class, div) { color: blue; }
```

### `:has()` 和 `:not()`

- `:has()` 的优先级计算与 `:is()` 相同，取参数中最高优先级
- `:not()` 不贡献自身优先级，但参数中参数选择器正常计入

```css
/* 优先级: 0,1,1,0 — 包含 #id 的选择器 */
div:has(#header) { color: red; }

/* 优先级: 0,1,1,1 — :not() 内部的选择器正常计入 */
div:not(.hidden)#nav { color: blue; }
```

## 样式层叠的来源对比

| 来源 | 优先级层级 | 说明 |
|------|-----------|------|
| 浏览器默认样式 | 最低 | 浏览器内置的 user agent stylesheet |
| 用户普通样式 | 低 | 用户通过浏览器设置的自定义样式 |
| 作者普通样式 | 中 | 开发者编写的 CSS |
| 作者 `!important` | 高 | 开发者标记的覆盖性样式 |
| 用户 `!important` | 最高 | 用户标记的覆盖性样式 |

## 优先级与层叠层（`@layer`）

使用 `@layer` 时，同一层内的规则按优先级计算，但不同层之间按层的定义顺序决定。后定义的层优先级更高：

```css
@layer base {
  #header { color: red; } /* 优先级高，但在更低的层中 */
}

@layer theme {
  .text-blue { color: blue; } /* 优先级低，但在更高的层中 */
}

/* 最终 text-blue 生效，因为 theme 层优先级高于 base 层 */
```

## 浏览器如何解析匹配规则

浏览器在匹配元素时，会经历以下步骤：

1. 收集所有应用到该元素的 CSS 规则
2. 按来源和 `!important` 分桶
3. 按优先级排序
4. 同优先级按出现顺序排序（后出现的覆盖先出现的）
5. 选出最终生效的属性值

**关键点**：优先级是浏览器解决声明冲突的核心依据，理解它有助于编写可维护、可预测的样式代码。
