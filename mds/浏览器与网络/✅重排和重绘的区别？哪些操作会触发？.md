# ✅重排和重绘的区别？哪些操作会触发？

# 典型回答

**重排（Reflow/Relayout）** 是指当 DOM 元素的几何属性（尺寸、位置等）发生变化时，浏览器需要重新计算元素的几何位置和尺寸，并重新构建布局树的过程。重排的代价很高，因为它会导致页面部分或全部布局重新计算。

**重绘（Repaint）** 是指当元素的外观属性（颜色、背景、可见性等）发生变化，但不影响布局时，浏览器只需要重新绘制元素的过程。重绘的代价比重排小，因为它不需要重新计算布局。

两者的核心区别在于：

| 对比维度 | 重排（Reflow） | 重绘（Repaint） |
|---------|---------------|-----------------|
| 触发条件 | 元素几何属性变化 | 元素外观属性变化 |
| 性能影响 | 大（需要重新计算布局） | 较小（不需要布局计算） |
| 涉及阶段 | Layout -> Paint -> Composite | Paint -> Composite |
| 影响范围 | 可能波及子元素和祖先元素 | 仅影响当前元素 |

浏览器会通过**渲染队列机制**来优化重排重绘，将多次变更合并为一次处理。但如果强制读取某些属性值（如 `offsetHeight`、`scrollTop` 等），会立即强制清空渲染队列，导致提前重排。

# 扩展知识

### 触发重排的常见操作

```javascript
// 1. 元素几何属性变化
element.style.width = '200px';
element.style.height = '300px';
element.style.margin = '20px';
element.style.padding = '10px';
element.style.border = '1px solid red';

// 2. DOM 树结构变化（增删改元素）
document.body.appendChild(newElement);
parentElement.removeChild(childElement);
element.innerHTML = '<div>新内容</div>';

// 3. 获取某些属性（强制同步布局）
const height = element.offsetHeight;      // 强制重排
const width = element.offsetWidth;        // 强制重排
const top = element.getBoundingClientRect().top; // 强制重排

// 4. 窗口尺寸变化
window.addEventListener('resize', () => {
  // 整个页面的布局需重新计算
});

// 5. 字体大小变化或字体加载
element.style.fontSize = '20px';

// 6. 激活 CSS 伪类（如 :hover）
// 7. 计算 offset 相关属性：offsetTop, offsetLeft, offsetParent
// 8. 计算 scroll 相关属性：scrollTop, scrollLeft, scrollWidth, scrollHeight
// 9. 计算 client 相关属性：clientTop, clientLeft, clientWidth, clientHeight
```

### 触发重绘但不触发重排的属性

```css
/* 这些属性变化只会导致重绘，不会触发重排 */
color             /* 文字颜色 */
background-color  /* 背景颜色 */
background-image  /* 背景图片 */
box-shadow        /* 阴影效果（仅部分情况） */
outline           /* 轮廓 */
border-color      /* 边框颜色 */
border-radius     /* 圆角（仅视觉效果） */
visibility        /* 可见性：hidden只触发重绘，不触发重排 */
text-decoration   /* 文本装饰 */
```

### 渲染队列机制

浏览器将连续的 DOM 变更放入渲染队列，在特定时机统一刷新：

```javascript
// 下面连续多次修改，浏览器会合并为一次重排
element.style.width = '100px';
element.style.height = '200px';
element.style.margin = '10px';
// 实际上只触发一次重排

// 强制清空渲染队列的操作
element.style.width = '100px';
const height = element.offsetHeight; // ← 这里强制触发重排！
element.style.height = '200px';
const width = element.offsetWidth;   // ← 再次强制触发重排！
// 结果：触发了两次重排（本可以合并成一次）
```

### 性能优化策略

```javascript
// 1. 批量修改样式（使用 class 代替内联样式）
// ❌ 不推荐
element.style.padding = '10px';
element.style.margin = '5px';
element.style.border = '1px solid';

// ✅ 推荐：使用 CSS class
element.classList.add('box-style');

// 2. 使用 DocumentFragment 批量操作 DOM
// ❌ 不推荐：多次插入触发多次重排
for (let i = 0; i < 100; i++) {
  const li = document.createElement('li');
  li.textContent = `Item ${i}`;
  ul.appendChild(li); // 每次循环都触发重排
}

// ✅ 推荐：使用 DocumentFragment
const fragment = document.createDocumentFragment();
for (let i = 0; i < 100; i++) {
  const li = document.createElement('li');
  li.textContent = `Item ${i}`;
  fragment.appendChild(li);
}
ul.appendChild(fragment); // 只触发一次重排

// 3. 使用 display: none 离线操作
// ✅ 推荐：离线修改后再显示
element.style.display = 'none';
// ... 进行多次样式修改（不触发重排）
element.style.width = '200px';
element.style.height = '300px';
element.style.padding = '20px';
element.style.display = 'block'; // 只触发一次重排

// 4. 缓存布局信息
// ❌ 不推荐：多次读取引起多次强制重排
for (let i = 0; i < items.length; i++) {
  items[i].style.width = container.offsetWidth + 'px';
  // 每次循环都触发一次重排
}

// ✅ 推荐：先缓存，后使用
const containerWidth = container.offsetWidth; // 读取一次
for (let i = 0; i < items.length; i++) {
  items[i].style.width = containerWidth + 'px';
} // 后续修改不会触发强制重排
```

### CSS 动画性能对比

```css
/* ❌ 不推荐：会触发 Layout 的动画属性 */
@keyframes bad-animation {
  from { left: 0; top: 0; }
  to   { left: 100px; top: 100px; }
}

/* ✅ 推荐：仅触发 Composite 的动画属性 */
@keyframes good-animation {
  from { transform: translate(0, 0); }
  to   { transform: translate(100px, 100px); }
}

/* ✅ 推荐：使用 opacity 做淡入淡出 */
@keyframes fade-in {
  from { opacity: 0; }
  to   { opacity: 1; }
}
```

| 动画属性 | 触发阶段 | 建议 |
|---------|---------|------|
| `left/top/margin` | Layout + Paint + Composite | 不推荐 |
| `width/height` | Layout + Paint + Composite | 不推荐 |
| `transform` | Composite | 推荐 |
| `opacity` | Composite | 推荐 |
| `background-color` | Paint + Composite | 谨慎使用 |
| `color` | Paint + Composite | 谨慎使用 |
